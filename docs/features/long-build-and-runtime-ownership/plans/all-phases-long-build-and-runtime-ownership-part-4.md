## Phase: long-build-and-runtime-ownership P4 — Torn-build atomic-promotion + `--cycle-begin` git-consistency recovery (M5 Detect)

**Goal:** The detect-and-recover net — Atomic Artifact Promotion (staging dir + `os.replace` on `exit(0)`) so a torn build never corrupts the production artifact, plus a `--cycle-begin` git-consistency check that removes a pre-boot `.git/index.lock` and `git clean`s the staging dir, composed with (NOT duplicating) the existing `--cycle-end` friction detector.

**Entry criteria:** Part 3 complete (`run_transient_build` — the build target promotion wraps); the existing `--cycle-begin`/`--cycle-end` bracket + `detect_cycle_bracket_friction`/`cycle_end_friction_check`.

**SPEC.md references:** `## Technical Design` → M5 Detect (Atomic Artifact Promotion + `--cycle-begin` git-consistency); LD4. Validation row "No orphaned/corrupt artifact on a mid-build tear".

### WU-1 — `promote_artifact_atomically`

**Scope (PHASES deliverable):** "`promote_artifact_atomically(staging_dir, final_dir, *, exit_code, replace=None) -> {promoted: bool, reason}` … `os.replace()` the staging artifact into the final path ONLY when `exit_code == 0`; a non-zero exit leaves the production artifact untouched. Injectable `replace` for hermetic tests."

**TDD:** yes.

**Files to create/modify:**
- `user/scripts/lazy_core.py` — add `promote_artifact_atomically(...)` (net-new; composes around the part-3 `run_transient_build` [VERIFY: `grep -n "def run_transient_build" user/scripts/lazy_core.py`] target).
- `user/scripts/test_lazy_core.py` — promotion fixtures.

**Test files:** `user/scripts/test_lazy_core.py`.

**Test expectations:**
- `exit_code == 0`: the injected `replace` is called staging→final → `{promoted: true}`; with a real temp-dir variant, the staging artifact ends up at `final_dir` and no partial file remains.
- `exit_code != 0`: `replace` is NEVER called, `final_dir` artifact untouched → `{promoted: false, reason: ...}` (no partial promotion).
- Atomicity ordering: assert the helper uses `os.replace` (atomic rename) not a copy-then-delete (inject `replace` and assert it is the single mutation).

**Implementation goal:** Make a torn build mathematically harmless — the production artifact is only ever swapped in atomically on a clean exit. LD4 detect-half.

**Spec requirements:** SPEC M5 Detect (Atomic Artifact Promotion); LD4.

**Batch:** 1.

### WU-2 — `--cycle-begin` git-consistency reconciliation + friction-detector composition

**Scope (PHASES deliverables):** "`--cycle-begin` git-consistency reconciliation … detect a pre-boot `.git/index.lock` (creation time older than the orchestrator/run-marker boot stamp) → remove it and `git clean -fdx` the staging dir; record the reconciliation. Best-effort + fail-open." + "Integration with the `--cycle-end` friction detector: the reconciliation is recorded so a torn-build delta neutralized at `--cycle-begin` does not subsequently false-trip `unexpected-commits`/`cycle-bracket-break` at `--cycle-end`."

**TDD:** yes.

**Files to create/modify:**
- `user/scripts/lazy_core.py` — add the reconciliation helper; compose with `detect_cycle_bracket_friction` [VERIFY: `grep -n "def detect_cycle_bracket_friction" user/scripts/lazy_core.py`] / `cycle_end_friction_check` [VERIFY: `grep -n "def cycle_end_friction_check" user/scripts/lazy_core.py`] (do NOT duplicate the detector — compose).
- `user/scripts/lazy-state.py` — wire the reconciliation into the `--cycle-begin` handler [VERIFY: `grep -n "cycle-begin" user/scripts/lazy-state.py`]; add `--test` fixtures. `bug-state.py` inherits via shared `lazy_core` (audited by `lazy_parity_audit.py`).
- `user/scripts/test_lazy_core.py` — reconciliation + composition fixtures.

**Test files:** `user/scripts/test_lazy_core.py` + in-file `lazy-state.py --test`.

**Test expectations:**
- Stale pre-boot `index.lock` (creation time < boot stamp, in a temp git tree) → removed + staging dir `git clean`ed; reconciliation recorded.
- Fresh/own lock (creation time >= boot stamp) → preserved (NOT removed) — never clobber a live git op.
- No lock / non-git tree → no-op, never raises (fail-open; the `--cycle-begin` write must always proceed).
- Composition: a torn-build delta neutralized at `--cycle-begin` does NOT subsequently cause `detect_cycle_bracket_friction` to report `unexpected-commits`/`cycle-bracket-break` at `--cycle-end` (assert against the real `detect_cycle_bracket_friction` — the reconciliation provably does not introduce a false positive).

**Implementation goal:** Compose the git-consistency reconciliation into the EXISTING cycle bracket so a torn-mid-build uncommitted delta is neutralized before the next cycle, without introducing a friction false-positive. LD4 detect-half (git-consistency).

**Spec requirements:** SPEC M5 Detect (`--cycle-begin` `.git/index.lock` / `git clean`); LD4; "Integrates with the existing `--cycle-end` friction detector".

**Batch:** 2.
