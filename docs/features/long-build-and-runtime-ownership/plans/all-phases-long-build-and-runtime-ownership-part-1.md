### WU-1 — `spawn_detached` cross-platform wrapper + breakaway-denied fallback

**Scope (PHASES deliverable):** "`spawn_detached(cmd, *, cwd, spawn=None, platform=None) -> {pid, start_time}` … Windows branch (`DETACHED_PROCESS | CREATE_NEW_PROCESS_GROUP | CREATE_BREAKAWAY_FROM_JOB`) with `try/except OSError` → plain `DETACHED_PROCESS | CREATE_NEW_PROCESS_GROUP` fallback; POSIX/WSL branch (`start_new_session=True`, wrapped in `systemd-run --user --scope --quiet --same-dir` when available, `setsid` + `nohup` keep-alive fallback). `PR_SET_PDEATHSIG` deliberately NOT used."

**TDD:** yes.

**Files to create/modify:**
- `user/scripts/lazy_core.py` — add `spawn_detached(...)` (net-new; sits alongside existing `ensure_runtime` [VERIFY: `grep -n "^def ensure_runtime" user/scripts/lazy_core.py`]).
- `user/scripts/test_lazy_core.py` — add characterization tests for both OS branches via injected `spawn`/`platform`.

**Test files:** `user/scripts/test_lazy_core.py` (extend; this file characterizes `lazy_core` helpers [VERIFY: `ls user/scripts/test_lazy_core.py`]).

**Test expectations:**
- Windows branch (injected `platform="win32"`, injected `spawn` capturing creationflags): asserts the first call carries `DETACHED_PROCESS | CREATE_NEW_PROCESS_GROUP | CREATE_BREAKAWAY_FROM_JOB`; on injected `OSError` (simulating `ERROR_ACCESS_DENIED`), asserts a second call WITHOUT `CREATE_BREAKAWAY_FROM_JOB` (the fallback), and that the function returns `{pid, start_time}` from the successful fallback.
- POSIX branch (injected `platform="linux"`): asserts `start_new_session=True`; when `systemd-run` is available (injected `which`), asserts the command is wrapped; when unavailable, asserts the `setsid` + keep-alive fallback path.
- A test asserting `PR_SET_PDEATHSIG` is NEVER set (grep the implementation / assert the spawn kwargs omit it).
- Returns `{pid: int, start_time: float|None}` — `start_time` may be filled by WU-2's `kernel_start_time` (inject a stub here so WU-1 is independently testable).

**Implementation goal:** A single cross-platform `spawn_detached` that escapes the Windows Job-Object `KILL_ON_JOB_CLOSE` reaping (breakaway, with the OSError fallback) and the POSIX/WSL idle-VM reaping (`systemd-run --user --scope`, setsid fallback), returning the child's `{pid, start_time}`. Injectable `spawn`/`platform` keep it hermetic — mirror the injected-callable pattern of `ensure_runtime` [VERIFY: `grep -n "def ensure_runtime" user/scripts/lazy_core.py`].

**Spec requirements:** SPEC M2 + LD6 (exact creationflags, the OSError fallback, `systemd-run`/setsid fallback, no `PR_SET_PDEATHSIG`).

**Batch:** 1.

### WU-2 — `kernel_start_time(pid)` temporal-identity extraction

**Scope (PHASES deliverable):** "`kernel_start_time(pid, *, platform=None) -> float | None` … Windows `ctypes`→`kernel32.GetProcessTimes`; POSIX/WSL `/proc/[pid]/stat` field 22 → epoch via `SC_CLK_TCK`. Best-effort, never raises (None on any error)."

**TDD:** yes.

**Files to create/modify:**
- `user/scripts/lazy_core.py` — add `kernel_start_time(...)` (net-new).
- `user/scripts/test_lazy_core.py` — parse fixtures.

**Test files:** `user/scripts/test_lazy_core.py`.

**Test expectations:**
- POSIX: feed a synthetic `/proc/[pid]/stat` line (field 22 = a known clock-tick value); assert the function converts ticks → epoch seconds via `SC_CLK_TCK` (inject the boot-time + clk-tck constants so the math is deterministic).
- Windows: inject a stubbed `GetProcessTimes` returning a known FILETIME; assert conversion to a Unix epoch float.
- Error path: an unreadable `/proc` entry / a stub raising → returns `None` (never raises).

**Implementation goal:** Stdlib-only kernel start-time extraction (the temporal-identity half of LD1) — the PID-reuse defense. Best-effort, None on any error so the DEAD/HIJACKED classification (part 2) consumes None cleanly.

**Spec requirements:** SPEC M1 (`start_time` field), LD6 (extraction mechanism).

**Batch:** 1.

### WU-3 — `.runtime.lock.json` read/write + new config keys

**Scope (PHASES deliverable):** "`write_runtime_lock(repo_root, *, pid, start_time, port, artifact_hash, controller_session_id)` and `read_runtime_lock(repo_root) -> dict | None` … atomic temp-file `os.replace` write of `.runtime.lock.json` at the project root with the five LD1 fields; best-effort read returns None on missing/corrupt." + "New config keys in `_ENSURE_RUNTIME_DEFAULT_CONFIG` (lock filename, port)."

**TDD:** yes.

**Files to create/modify:**
- `user/scripts/lazy_core.py` — add `write_runtime_lock`/`read_runtime_lock`; extend `_ENSURE_RUNTIME_DEFAULT_CONFIG` [VERIFY: `grep -n "_ENSURE_RUNTIME_DEFAULT_CONFIG: dict" user/scripts/lazy_core.py`] with `lock_filename` + `port` keys (do NOT hard-code into the flow).
- `user/scripts/test_lazy_core.py` — round-trip + corrupt-read fixtures.

**Test files:** `user/scripts/test_lazy_core.py`.

**Test expectations:**
- Round-trip: write a lock with all five LD1 fields (`controller_session_id`, `pid`, `start_time`, `port`, `artifact_hash`) → read it back → all fields equal.
- Atomicity: write uses a temp file + `os.replace` (assert no partial file on a simulated mid-write failure, mirroring the `_increment_tally` / cycle-marker atomic-write pattern already in the tree).
- Corrupt/missing read → `None` (never raises).
- New config keys present in `_ENSURE_RUNTIME_DEFAULT_CONFIG` and read via `cfg[...]`, not literals.

**Implementation goal:** The `.runtime.lock.json` sentinel persistence (LD1) — atomic, best-effort, schema-exact. The lock path comes from the config dict so the AlgoBooth port/filename stay parameterized.

**Spec requirements:** SPEC M1 sentinel field table; LD1.

**Batch:** 2.

### WU-4 — `verify_runtime_ownership` predicate

**Scope (PHASES deliverable):** "`verify_runtime_ownership(lock, *, live_session_id, kernel_start_time_fn) -> bool` … True iff recorded `start_time` matches the kernel-reported start_time for `lock['pid']` AND `controller_session_id == live_session_id`; defeats PID-reuse + foreign-controller cases."

**TDD:** yes.

**Files to create/modify:**
- `user/scripts/lazy_core.py` — add `verify_runtime_ownership(...)` (consumes the WU-3 lock dict shape + WU-2 `kernel_start_time`).
- `user/scripts/test_lazy_core.py` — verification matrix.

**Test files:** `user/scripts/test_lazy_core.py`.

**Test expectations (the LD1 verifiability matrix):**
- Match: recorded `start_time` == injected `kernel_start_time_fn(pid)` AND `controller_session_id` == `live_session_id` → `True`.
- Divergent start_time (PID reused by a foreign process): injected `kernel_start_time_fn` returns a different value → `False`.
- Foreign controller: `controller_session_id` != `live_session_id` → `False`.
- Missing PID (process dead): injected `kernel_start_time_fn` returns `None` → `False`.

**Implementation goal:** The verifiability predicate (LD1) — "200 on /health" is NOT proof of ownership; only the `(start_time, controller_session_id)` match is. This is the Identity-phase input part 2's reworked `ensure_runtime` consumes.

**Spec requirements:** SPEC M1 "Ownership is **verifiable**"; LD1; Validation row "Ownership is verifiable, not just health=200".

**Batch:** 3.
