### WU-7 — queue_writer + POST route + run-marker refusal

**Scope (PHASES Phase 4 deliverables):** `queue_writer.py`, `server.py` `POST /api/queue` + run-marker detection + `queue_locked` flag.
**TDD:** yes.
**Files to create/modify:**
- `user/scripts/pipeline_visualizer/queue_writer.py` — new: permutation-validated atomic reorder write with AV-lock retry
- `user/scripts/pipeline_visualizer/server.py` — modify: `POST /api/queue` route; expose `queue_locked` on `/api/state`; detect run-marker via `lazy_core.read_run_marker()` [VERIFY: grep -n "def read_run_marker" user/scripts/lazy_core.py]
**Test files:**
- `user/scripts/test_pipeline_visualizer.py` (extend)
**Test expectations:**
- Permutation validation: a new order that adds/drops/dupes an ID is rejected (no write); a true permutation is accepted.
- Atomic round-trip: reorder persists; resulting `queue.json` is valid JSON written via temp + `os.replace` with `indent=2` + trailing newline, matching `lazy-state.py`'s `_atomic_write` convention so `/lazy` reads it cleanly [VERIFY: grep -n "_atomic_write(queue_path" user/scripts/lazy-state.py].
- AV-lock retry: monkeypatch `os.replace` to raise `PermissionError` with `winerror` 32 twice then succeed → write completes within ≤3 tries (assert call count); exhausting all 3 retries → raises/returns a clear error (POST → 503).
- Run-marker refusal: with `read_run_marker()` returning a marker (monkeypatched or a real `--run-start` fixture), `POST /api/queue` → 409 and `queue.json` is byte-identical; `/api/state` reports `queue_locked: true`. With no marker → 200 + write succeeds.
**Implementation goal:** `queue_writer.py` reads `queue.json`, validates the posted order is a permutation of existing IDs, reorders the array, writes temp file + `os.replace` wrapped in a 3×/50ms retry catching `PermissionError` for `[WinError 5]`/`[WinError 32]`. `server.py` gates the POST on `lazy_core.read_run_marker()` (returns 409 + `queue_locked` while present); refusal is the floor (Decision 6/11) — intent-log is explicitly v2, NOT built here.
**Spec requirements:** Decisions 6 + 11; Validation "Reorder persists (idle)", "Reorder refused (run active)", "Write survives AV lock".
**Batch:** 1.
