## Phase: lazy-pipeline-visualizer P1 — Backend read layer + curated-stage mapping

**Goal:** Stand up a stdlib `ThreadingHTTPServer` that shells the existing state scripts, attaches `curated_stage`, reads `queue.json`/`leases.json`/`ROADMAP.md`, caches the probe (TTL 2.0s), and serves `/api/state` + `/api/queue` — closing the full process→serialization→HTTP loop end-to-end.

**Entry criteria:** None (first phase).

**SPEC.md references:** "Technical Design", "Curated-node rollup (display mapping)", "Implementation Phases — Phase 1", Validation rows "Token on correct stage", "Cache debounces probe", "Backend stays responsive", "Fleet reflects leases".

### WU-1 — curated_stage rollup + leases freshness

**Scope (PHASES Phase 1 deliverables):** `curated_stage.py`, `leases.py`.
**TDD:** yes.
**Files to create:**
- `user/scripts/pipeline_visualizer/__init__.py`
- `user/scripts/pipeline_visualizer/curated_stage.py`
- `user/scripts/pipeline_visualizer/leases.py`
**Test files:**
- `user/scripts/test_pipeline_visualizer.py` (new — this WU seeds it)
**Test expectations:**
- `curated_stage(current_step, terminal_reason, pipeline)` returns the correct curated node for EVERY literal in the SPEC "Curated-node rollup" table, for both `pipeline="feature"` and `pipeline="bug"`. Assert each side-state (`blocked`→`Blocked`, `needs-input`/`needs-research`/`needs-spec-input`→`Needs-input`, cloud/device deferral terminals→`Deferred`). Assert an unknown/`None` step falls back to a documented default (e.g. `Spec` for a queued-but-unstarted item) — pick the safe default and assert it.
- `lease_view(entry, now)` computes `heartbeat_fresh` using `lazy_coord.py`'s rule: fresh iff `heartbeat_epoch + ttl_seconds >= now`. Assert the boundary: exactly-at-expiry is fresh, one second past is stale. Assert it surfaces `{wi_id, worker_pid, worktree_slot, term_token, heartbeat_fresh, age_seconds}`.
**Implementation goal:** `curated_stage.py` is a pure mapping function whose table mirrors the SPEC rollup [VERIFY: grep -n "Curated-node rollup" docs/features/lazy-pipeline-visualizer/SPEC.md]; it does NOT shell or re-infer anything. `leases.py` reuses `lazy_coord.py`'s freshness arithmetic [VERIFY: grep -n "heartbeat_timestamp\|ttl_seconds" user/scripts/lazy_coord.py] — import its `_parse_iso` helper if cleanly importable, else replicate the exact ISO-8601 'Z' parse and document why.
**Spec requirements:** SPEC "Curated-node rollup (display mapping)" table; Open Questions §1 (RESOLVED) leases schema; SPEC color/shape table for the side-state names.
**Batch:** 1.

### WU-2 — probe + TTL cache

**Scope (PHASES Phase 1 deliverables):** `probe.py`, `cache.py`.
**TDD:** yes.
**Files to create:**
- `user/scripts/pipeline_visualizer/probe.py`
- `user/scripts/pipeline_visualizer/cache.py`
**Test files:**
- `user/scripts/test_pipeline_visualizer.py` (extend)
**Test expectations:**
- `cache`: a `TtlCache(ttl=2.0, clock=fake_clock)` with a counted producer fn; N concurrent/sequential reads within one TTL window invoke the producer exactly once; a read after `ttl` elapses re-invokes. Use a monkeypatched/injected clock — NO real sleeps. Assert the double-checked-lock path is exercised (two reads at the same fake time → 1 producer call).
- `probe`: parse a captured-real `lazy-state.py` JSON fixture into the per-item dict; assert it attaches `curated_stage` (from WU-1) and preserves `feature_id`/`current_step`/`terminal_reason`. Assert a malformed/empty script output is handled (item flagged, not a crash). Assert `read_queue(path)` returns the array order and `read_leases(path)` returns the lease views; missing files → empty (not error).
**Implementation goal:** `probe.py` shells `lazy-state.py --repo-root <root> --feature-id <id>` and `bug-state.py --bug-id <id>` per queue item via `subprocess.run(..., capture_output=True, text=True)` then `json.loads` [VERIFY: grep -n '"current_step"' user/scripts/lazy-state.py] [VERIFY: grep -n '"current_step"' user/scripts/bug-state.py]; reads `docs/features/queue.json`, `docs/bugs/queue.json`, `leases.json`, `ROADMAP.md`. Returns one aggregate dict `{features: [...], bugs: [...], leases: [...], roadmap: {...}, server_time: "<ISO>"}`. `cache.py` is a read-through TTL cache (TTL 2.0s as a single named constant) with a double-checked `threading.Lock`. State is NEVER re-inferred — the probe only shells + parses.
**Spec requirements:** SPEC "Technical Design" (state never re-inferred; ThreadingHTTPServer + TTL 2.0s cache); Validation "Cache debounces probe (Heavy probe runs ≤ once / 2.0s)".
**Batch:** 2.

Reachability smoke for the new API surface lives in Runtime Verification below.
