## Phase: lazy-pipeline-visualizer P2 — Static frontend shell + read-only three-pane render

**Goal:** Serve a browser frontend that renders the three panes from live `/api/state`, with headless→preset layout bootstrap, colorblind-safe encoding, a polling loop, and the live-dot + Connection-Lost guard. Read-only — no traversal animation (Phase 3), no reorder writes (Phase 4).

**Entry criteria:** Part-1 complete — `/api/state` + `/api/queue` JSON contract and `ThreadingHTTPServer` exist.

**SPEC.md references:** "User Experience — Three-pane layout", "Color & shape encoding", "Implementation Phases — Phase 2", Validation "Token on correct stage", Decision 7 (Cytoscape no build step), Decision 12 (poll 2.5s).

### WU-4 — static-asset serving + vendored libs + server static-root

**Scope (PHASES Phase 2 deliverables):** static-asset serving, vendored `cytoscape.umd.js` / `dagre.js` / `cytoscape-dagre.js`, `server.py` static-root wiring.
**TDD:** yes (serving is HTTP-assertable).
**Files to create/modify:**
- `user/scripts/pipeline_visualizer/static/cytoscape.umd.js` — new (vendored UMD, no build step) [VERIFY: SPEC Decision 7 "standalone UMD/ES `<script>`"]
- `user/scripts/pipeline_visualizer/static/dagre.js` — new (vendored)
- `user/scripts/pipeline_visualizer/static/cytoscape-dagre.js` — new (vendored adapter)
- `user/scripts/pipeline_visualizer/server.py` — modify: resolve static-root to the `static/` dir; defer non-API GETs to `SimpleHTTPRequestHandler` super [VERIFY: grep -n "SimpleHTTPRequestHandler" user/scripts/pipeline_visualizer/server.py]
**Test files:**
- `user/scripts/test_pipeline_visualizer.py` (extend)
**Test expectations:**
- `GET /` → 200 + `text/html` (serves `index.html`); `GET /static/app.js` → 200 + a JS content-type; `GET /static/cytoscape.umd.js` → 200. Assert API routes still win over static (regression: `/api/state` still 200 JSON, not treated as a file path).
- Path-traversal guard: `GET /static/../server.py` does NOT serve the source file (SimpleHTTPRequestHandler normalizes, but assert it).
**Implementation goal:** wire the handler's static-root to `pipeline_visualizer/static/`; API routes are matched before the static fallthrough. Vendored libs are committed as-is (no build, no minify step required) per Decision 7.
**Spec requirements:** Decision 7; SPEC Technical Design backend bullet ("static frontend assets deferred to super").
**Batch:** 1.

### WU-5 — three-pane shell + poll loop + encoding

**Scope (PHASES Phase 2 deliverables):** `index.html`, `app.js`, `styles.css`, layout bootstrap, color/shape encoding, queues/fleet/triage panes, live dot + Connection-Lost guard.
**TDD:** no (browser UI — no DOM harness in this repo; validated via manual checklist). The serving of these assets is covered by WU-4's automated tests.
**Files to create:**
- `user/scripts/pipeline_visualizer/static/index.html` — three-pane shell (hero graph region / bottom-left queues / bottom-right fleet) + triage strip + footer live dot
- `user/scripts/pipeline_visualizer/static/app.js` — poll loop (interval 2.5s) `fetch('/api/state')`; render panes; on fail/timeout flip live dot red + dim + "Connection Lost" banner
- `user/scripts/pipeline_visualizer/static/styles.css` — three-pane grid + colorblind-safe encoding styles
- `docs/features/lazy-pipeline-visualizer/MANUAL_TESTING.md` — new browser smoke checklist (behavior-focused, per `/write-manual-testing-doc` conventions)
**Implementation goal:**
- Layout bootstrap: build the stage-node graph, run `dagre rankDir:'LR'` in a HEADLESS Cytoscape instance, extract settled `(x,y)` for both tracks, then render the live canvas with the `preset` layout (immutable positions). Never run a layout on poll; never use `cola`/`cose`/`elk` [VERIFY: SPEC "Layout:" Technical Design bullet]. Expose the stage→`(x,y)` map as a module-level object (Phase 3 animates against it).
- Tokens are flat peers at higher z-index (NOT compound nodes) — render each item at its `curated_stage` coordinate. No animation yet (full re-render per poll is acceptable in this phase).
- Color + shape encoding from the SPEC table (Pending gray/hollow, Running blue/▶, Complete green/✓, Needs-Input orange/hexagon, Blocked red/octagon, Deferred purple/dashed-ghost) [VERIFY: grep -n "Color & shape encoding" docs/features/lazy-pipeline-visualizer/SPEC.md].
- Queues pane: two vertical lists (Features, Bugs) from `/api/queue` with tier/ad-hoc/stub badges. Static rows (drag is Phase 4).
- Fleet pane: `wt-NN` cards from `leases[]` (ID + shape/color + derived branch + heartbeat freshness + worker pid); empty when `leases[]` empty.
- Triage strip: lists every item whose `curated_stage` ∈ {Blocked, Needs-input, Deferred}.
- Live dot: green on 200 polls; on any poll fail/timeout → red + screen dim + "Connection Lost" banner within ≤1 poll interval. Never present stale-as-live.
**Spec requirements:** SPEC "User Experience" three-pane + glance test; "Color & shape encoding"; Decision 10 (two-track is wired visually in Phase 3, but the panes/encoding land here); Decision 12 (2.5s poll).
**Batch:** 2.
