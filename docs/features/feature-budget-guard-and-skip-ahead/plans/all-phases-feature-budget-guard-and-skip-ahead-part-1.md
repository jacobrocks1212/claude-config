- **Test files:** `user/scripts/test_lazy_core.py`.
- **Test expectations:** `compute_per_feature_ceiling(max_cycles, ready_queue_depth, override=None)` returns `override` when supplied; else `max(6, min(C * 4 // 10, (C // Q) * 2))` (integer floor division per Locked Decision 4). Boundary cases asserted: `Q_depth == 0` → 6 floor (no div-by-zero); `max_cycles=12, Q=2` → floor-of-6 honored; deep queue (e.g. `C=32, Q=10`) → 6 (the example in RESEARCH_SUMMARY); large `C` shallow `Q` → 40% cap arm wins. Pure + side-effect-free.
- **Implementation goal:** add the pure ceiling fn; add `--per-feature-cycle-cap <int>` to `lazy-state.py` argparse, thread it into `compute_state()` as an APPENDED keyword param `per_feature_cycle_cap: int | None = None` (positional callers unbroken — mirror the `--park-*` flag threading [VERIFY: grep -n "park_blocked: bool = False\|park_needs_input: bool = False" user/scripts/lazy-state.py]).
