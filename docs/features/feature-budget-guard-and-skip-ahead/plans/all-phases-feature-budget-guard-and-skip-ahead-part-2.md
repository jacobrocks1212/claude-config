### Phase: feature-budget-guard-and-skip-ahead P3 — Dependency-aware skip-ahead (two-key predicate)

**Goal:** Generalize the all-or-nothing research skip into a default-on, dependency-aware skip-ahead past a gated head, gated by a two-key readiness predicate (hard-dep absence AND `independent: true`), with a `--strict-research-halt` opt-out.
**Entry criteria:** Parts 1–2 complete (independent of the budget-guard logic, but follows the same flag-threading pattern).
**SPEC.md references:** Technical Design → Skip-ahead past a gated head → Readiness predicate (two-key); Locked Decisions 3 + 5; Validation Criteria rows "Skip-ahead advances onto independent item", "Unmarked item NOT skipped onto", "Downstream item NOT skipped onto", "All-gated terminal".

#### WU-1 — Two-key readiness predicates
- **Scope:** PHASES P3 deliverables "`parse_independent_marker`" + "`skip_ahead_ready`".
- **TDD:** yes — characterize both predicates directly in `test_lazy_core.py` (FAILING tests first).
- **Files to create/modify:** `user/scripts/lazy_core.py` (or co-locate in `lazy-state.py` if they need the local dep parser — see note), `user/scripts/test_lazy_core.py`.
- **Test files:** `user/scripts/test_lazy_core.py`.
- **Test expectations:**
  - `parse_independent_marker(spec_text, queue_entry)` → `True` for an explicit `independent: true` in SPEC frontmatter OR queue entry; `True` for the `no_shared_state: true` alias; `False` when absent (default). Deterministic on-disk read; no LLM judgment.
  - `skip_ahead_ready(candidate, gated_ids, ...)` → reuses `parse_dep_block` [VERIFY: grep -n "def parse_dep_block" user/scripts/lazy-state.py] to get deps, filters `kind == "hard"`, returns `False` if any hard-dep `feature_id` is in `gated_ids`; else requires `parse_independent_marker` truthy. Asserts: independent + no hard-dep-on-gated → True; hard-dep on a gated id (even if marked) → False; `soft`/`composes` dep on a gated id does NOT block; unmarked but dep-free → False.
- **Implementation goal:** add both predicates as pure (or near-pure) functions. NOTE: `parse_dep_block` currently lives in `lazy-state.py` (line ~831). Prefer placing `skip_ahead_ready` where it can call `parse_dep_block` directly — either co-located in `lazy-state.py`, or move/share `parse_dep_block` into `lazy_core` if Phase-4 parity wants `bug-state.py` to import it (the bug pipeline has no research gate, so a shared move is optional — keep the simplest placement that keeps both `--test` suites green).
- **Spec requirements:** Locked Decision 5 (two-key predicate, `independent: true` / `no_shared_state` marker, `soft`/`composes` non-blocking).
- **Batch:** 1.

#### WU-2 — Skip-ahead branch in `compute_state()` + `--strict-research-halt` flag
- **Scope:** PHASES P3 deliverables "`--strict-research-halt` arg" + "wire skip-ahead into the queue-selection loop".
- **TDD:** yes — author FAILING `--test` fixtures FIRST (skip onto independent; do NOT skip onto unmarked; do NOT skip onto downstream; `--strict-research-halt` restores legacy halt).
- **Files to create/modify:** `user/scripts/lazy-state.py` (arg + `compute_state()` branch), `user/scripts/bug-state.py` (mirror arg for parity — no-op there), `user/scripts/tests/baselines/{lazy-state,bug-state}-test-baseline.txt`.
- **Test files:** in-file `--test` harness in `user/scripts/lazy-state.py` (extend the existing `needs-research` fixture [VERIFY: grep -n 'elif name == "needs-research"' user/scripts/lazy-state.py]).
- **Test expectations:**
  - Default (no `--strict-research-halt`): with a research-gated head, a candidate that passes `skip_ahead_ready` IS dispatched as `current`; an unmarked-but-dep-free candidate is NOT (degrades to today's strict behavior for that item); a downstream candidate (hard dep on the gated head, even if marked) is NOT.
  - `--strict-research-halt` set: reproduces the legacy halt-on-first-gated-head (same terminal the pre-feature `--skip-needs-research`-absent path produces).
- **Implementation goal:** add `--strict-research-halt` to `lazy-state.py` argparse, thread as appended `compute_state()` kwarg `strict_research_halt: bool = False` (positional callers unbroken). In the queue-selection loop [VERIFY: grep -n "def compute_state" user/scripts/lazy-state.py], when the head candidate is gated (research-pending per the existing peek [VERIFY: grep -n "if skip_needs_research:" user/scripts/lazy-state.py], OR BLOCKED) and `--strict-research-halt` is NOT set: add the gated head to a `gated_ids` set + a surfaced `_GATED_HEADS` list and `continue` to the next candidate (mirror the `skip_needs_research` skip shape); dispatch the first candidate passing `skip_ahead_ready(candidate, gated_ids, ...)`. An unmarked/downstream candidate is NOT dispatched. Mirror the `--strict-research-halt` flag onto `bug-state.py` for parity (no-op: the bug pipeline has no research gate — document it).
- **Spec requirements:** Locked Decision 3 (default-on, `--strict-research-halt` opt-out); Validation Criteria rows for skip-onto-independent / not-onto-unmarked / not-onto-downstream.
- **Batch:** 1.
