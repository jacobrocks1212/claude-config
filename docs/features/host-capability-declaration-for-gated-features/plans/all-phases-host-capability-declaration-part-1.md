#### WU-1 — `_HOST_CAPABILITY_REGISTRY` + `parse_requires_host` + `unknown_capability_ids`

- **Scope:** Phase 1 deliverables — registry constant, `parse_requires_host`, `unknown_capability_ids`, and their tests.
- **TDD:** yes.
- **Files to create/modify:**
  - `user/scripts/lazy_core.py` — add the constant + helpers beside the existing `_INDEPENDENT_MARKER_*` / `parse_independent_marker` block [VERIFY: grep -n "def parse_independent_marker" user/scripts/lazy_core.py] (expected line ~9864).
  - `user/scripts/test_lazy_core.py` — add test functions + register in `_TESTS` [VERIFY: grep -n "_TESTS = \[" user/scripts/test_lazy_core.py] (expected line ~14532).
- **Test files:** `user/scripts/test_lazy_core.py`.
- **Test expectations:**
  - `parse_requires_host` reads a capability set from SPEC frontmatter only; from a `queue.json` entry only; from both (union).
  - Absent field ⇒ empty set (ungated) — the baseline-regression lock.
  - List value and a comma/space-separated string both parse to the same set (tolerant input).
  - The id-shape regex `^[a-z0-9][a-z0-9-]*$` rejects a malformed id (raise or drop — assert the chosen contract).
  - `unknown_capability_ids({"typo-cap"})` returns `{"typo-cap"}`; a registered id returns `set()`.
  - Module-load assertion: every `_HOST_CAPABILITY_REGISTRY` key is shape-valid.
- **Implementation goal:** `parse_requires_host(spec_text, queue_entry) -> set[str]` mirrors the two-source fenced-block walk of `parse_independent_marker` [VERIFY: grep -n "def parse_independent_marker" user/scripts/lazy_core.py]. `_HOST_CAPABILITY_REGISTRY` is a dict whose KEYS are the closed v1 vocabulary (seed `real-audio-device`, `zimtohrli-toolchain`, `gpu`); VALUES are probe-callable placeholders filled in WU-3. `unknown_capability_ids(required)` returns `required - set(_HOST_CAPABILITY_REGISTRY)`.
- **Spec requirements:** "Capability ids match `^[a-z0-9][a-z0-9-]*$` … drawn from the closed registry"; "absent/legacy ⇒ empty set (ungated)"; "An unregistered `requires_host:` id is a loud, immediate validation failure" (the helper that detects it — the action is Phase 4).

### Phase: host-capability-declaration P2 — Active-invocation probe primitives

**Entry criteria:** P1 complete (registry keys define which primitives are needed).
**SPEC.md references:** "The host-capability probe", "Probes use active invocation, not filesystem presence", Validation Criteria rows "Host probe is hermetic" + "Probe uses active invocation, not which()".

#### WU-2 — `probe_binary_capability` / `probe_env_capability` (active-invocation, injected)

- **Scope:** Phase 2 deliverables — the two probe primitives + the `\WindowsApps` alias-false-positive guard test.
- **TDD:** yes.
- **Files to create/modify:**
  - `user/scripts/lazy_core.py` — add `probe_binary_capability` / `probe_env_capability` near `ensure_runtime` [VERIFY: grep -n "def ensure_runtime" user/scripts/lazy_core.py] (expected line ~6457), reusing `_FALSY_ENV_VALUES` [VERIFY: grep -n "_FALSY_ENV_VALUES =" user/scripts/lazy_core.py] (expected line ~2166).
  - `user/scripts/test_lazy_core.py` — probe-primitive tests + `_TESTS` registration.
- **Test files:** `user/scripts/test_lazy_core.py`.
- **Test expectations:**
  - `probe_binary_capability(argv, run=<stub exit 0>)` ⇒ `True`; `run=<stub exit 1>` ⇒ `False`.
  - Alias-false-positive guard: an injected `run` simulating the `\WindowsApps` zero-byte stub (exit non-zero / would-hang) ⇒ `False` — assert the active-invocation contract, NOT a filesystem-presence check.
  - `probe_env_capability(var, environ={set})` ⇒ True; unset ⇒ False; falsy value (`"0"`, `""`, `"false"`) ⇒ False (via `_FALSY_ENV_VALUES`).
  - Hermetic: injected `run`/`environ` only — no real binary/env touched.
- **Implementation goal:** `probe_binary_capability(argv, *, run=None) -> bool` runs `argv` via injected `run` (default real `subprocess.run` with a short timeout, `capture_output=True`, `shell=False`) and returns `exit code == 0`. NEVER calls `shutil.which`/`os.path.exists`. `probe_env_capability(var_name, *, environ=None) -> bool` is truthy iff the var is set and its value is not in `_FALSY_ENV_VALUES`.
- **Spec requirements:** "A binary-capability probe MUST run the tool (`subprocess.run([tool, "--version"|"--help"])`) and check the exit code … It MUST NOT use `shutil.which()` / `os.path.exists()`"; the `\WindowsApps` App-Execution-Alias false-positive hazard.

### Phase: host-capability-declaration P3 — Host-present-set resolver + per-run cache

**Goal:** Compose the probe primitives into a single resolver returning the host's present-capability set, hermetic via injection, cached per run-marker in the per-repo keyed state dir.
**Entry criteria:** P1 (registry), P2 (primitives) complete.
**SPEC.md references:** "The host-capability probe" (cache-per-run), Locked Decision 1 (runtime probe is the source of truth), Validation Criteria "Host probe is hermetic".

#### WU-3 — `host_present_capabilities` resolver + per-run cache + production binding

- **Scope:** Phase 3 deliverables — the resolver, the per-run cache, the production registry→primitive binding, and tests.
- **TDD:** yes.
- **Files to create/modify:**
  - `user/scripts/lazy_core.py` — `host_present_capabilities(*, probes=None, cache=True) -> set[str]` + cache read/write under `claude_state_dir()` [VERIFY: grep -n "def claude_state_dir" user/scripts/lazy_core.py] (expected line ~7773); bind each `_HOST_CAPABILITY_REGISTRY` value to a Phase-2 primitive.
  - `user/scripts/test_lazy_core.py` — resolver + cache tests + `_TESTS` registration.
- **Test files:** `user/scripts/test_lazy_core.py`.
- **Test expectations:**
  - `host_present_capabilities(probes={"gpu": lambda: True, "zimtohrli-toolchain": lambda: False, ...})` ⇒ exactly the True-valued ids.
  - Cache writes once per run; a NEW run marker re-probes (assert via the `LAZY_STATE_DIR`-override hermetic seam).
  - No run marker ⇒ probes fresh (no cache).
  - Hermetic: injected `probes` only; no real `~/.claude/state/` touched (override `LAZY_STATE_DIR` to a temp dir).
- **Implementation goal:** For each registry id, evaluate its bound probe callable; return the present set. `probes` injects a `{capability-id: callable}` map (default: real production bindings) so `--test` passes a stub map — mirrors `ensure_runtime`'s "real defaults bound only when callables are None" [VERIFY: grep -n "def ensure_runtime" user/scripts/lazy_core.py]. Cache the present-set as JSON under `claude_state_dir()` keyed to the run-marker identity; cache-per-run, re-probe on a new marker; non-destructive read. Production binding: `real-audio-device` → `probe_env_capability("ALGOBOOTH_REAL_AUDIO_DEVICE")`; `zimtohrli-toolchain` → `probe_binary_capability([...])`; `gpu` → a documented probe. Keep AlgoBooth-specific argv config-overridable, not hard-coded into the resolver flow.
- **Spec requirements:** "The host's present-capability set is resolved by a new `lazy_core` helper with injected probe callables (so `--test` stays hermetic)"; "cache for the run, re-probe on a new run marker — cheapest correct option."
