#### WU-3 — Capability-miss skip branch + re-open + `host-capability-saturated` terminal + probe field

- **Scope:** the behavioral core in `compute_state` + the terminal + the orchestrator probe field, plus the four `--test` fixtures.
- **TDD:** yes.
- **Files to create/modify:**
  - `user/scripts/lazy-state.py` — in `compute_state` [VERIFY: grep -n "def compute_state" user/scripts/lazy-state.py], add the capability-miss branch as a SIBLING to the `not real_device` device-saturated skip [VERIFY: grep -n "device_saturated_skipped.append" user/scripts/lazy-state.py] (expected line ~1511): compute `missing`, on non-empty write `DEFERRED_REQUIRES_HOST.md`, append to a `host_saturated_skipped` list + a `_HOST_DEFERRED` accumulator, emit the per-probe diagnostic naming the missing ids, and `continue`. Add the `host-capability-saturated` terminal beside `device-queue-exhausted` [VERIFY: grep -n "device-queue-exhausted" user/scripts/lazy-state.py] (expected line ~1846) with the same precedence ordering. Add the probe field for the orchestrator (notification + live-queue skip), modeled on the budget-guard `_GATED_HEADS` surfacing [VERIFY: grep -n "_GATED_HEADS" user/scripts/lazy-state.py].
  - `user/scripts/test_lazy_core.py` / in-file fixtures + `user/scripts/tests/baselines/lazy-state-test-baseline.txt`.
- **Test files:** `user/scripts/test_lazy_core.py` / `lazy-state.py --test` fixtures.
- **Test expectations (the SPEC Validation Criteria rows):**
  - (a) **Composite AND:** two caps, one present one absent ⇒ `missing` is the absent cap ⇒ defers (subset-match; any miss ⇒ defer).
  - (b) **Re-open:** same feature with the cap now present (`missing` empty) ⇒ dispatched into runtime validation (no skip).
  - (c) **Saturated terminal:** every remaining feature gated on an absent cap ⇒ `host-capability-saturated` terminal (clean, NOT `all-features-complete`).
  - (d) **Ungated unaffected:** a feature with no `requires_host:` ⇒ byte-identical to baseline (the load-bearing baseline-regression guard).
- **Implementation goal:** the defer preserves the "`Complete` means fully validated" invariant — the feature is testable, just not on THIS host, so it re-opens on a capability-bearing host rather than being skipped (permanent waiver) or back-of-queued (no cross-host re-open). Re-open is no-special-case: an empty `missing` simply does not skip.
- **Spec requirements:** "In `compute_state()` queue selection, before dispatching the current feature's next runtime-verification sub-skill, compute `missing = feature.requires_host - host.present`. On non-empty `missing`, take the Decision-2 action: defer-to-capability-host"; the notification format `host-capability miss — <feature-id> requires <cap-id> (absent on this host); deferred to capability-host`.
- **Batch:** 2.
