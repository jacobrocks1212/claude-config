### Phase: harness-change-canary-rollback P2 — Watcher: windows, attribution, tripwire

**Goal:** `efficacy-eval.py --canary` accrues each open canary's window at every run boundary, applies the D2 bands and D3 surface-based attribution, detects trips, and degrades honestly on missing data — never blocking a run.
**Entry criteria:** Part 1 landed (records carry the `canary:` sub-map with `surfaces`/`window_runs`/`status: open`).
**SPEC.md references:** D2 (window sizing + tripwire bands), D3 (incident attribution), D6 (watcher = evaluator mode); Technical Design "Watcher" + "Incident sources (v1)"; Validation Criteria "Band tripwire", "Incident tripwire + attribution", "Never blocks a run".

#### WU-4 — `--canary` mode scaffold + per-run window accrual + no-data handling

- **Scope:** PHASES.md P2 deliverables: the `--canary` branch in `efficacy-eval.py` `main()` reusing `_enumerate_records` (filtered to `canary.status: open`), the telemetry `run_ids` accrual, `_write_record`; D2 window accrual (next 10 runs / 30-day ceiling; per-record overridable); honest no-data (`closed-clean (no-data)` for a zero-observable-run window; never errors).
- **TDD:** yes
- **Files to create/modify:** `user/scripts/efficacy-eval.py`
- **Test files:** `user/scripts/test_efficacy_eval.py`
- **Test expectations:**
  - `efficacy-eval.py --canary` enumerates only records with `canary.status: open` (a `closed-clean`/`tripped`/no-canary record is skipped).
  - Window accrual counts post-ship distinct `run_id`s up to the 10-run window; the 30-day ceiling closes a window even with < 10 runs.
  - An unreadable/absent telemetry fixture → the window accrues nothing this run and the invocation returns exit 0 (never raises).
  - A matured window with zero observable runs stamps `closed-clean (no-data)`.
- **Implementation goal:** Add a `--canary` argument + branch to `main()` [VERIFY: grep -n "def main" user/scripts/efficacy-eval.py] reusing `_enumerate_records` and the `run_ids` accrual already built for the efficacy review [VERIFY: grep -n "def _enumerate_records\|def _write_record" user/scripts/efficacy-eval.py] and `lazy_core.read_intervention_telemetry` [VERIFY: grep -n "read_intervention_telemetry" user/scripts/efficacy-eval.py]. Keep the two cadences behind a clean `--canary` boundary (separate helpers). Fail-open on every read path.
- **Spec requirements:** SPEC D2 Resolution; D6-A (evaluator mode, every run while open); "Failure modes" (never blocks a run).
- **Batch:** 1

#### WU-5 — D2 tripwire bands + D3 surface-based incident attribution

- **Scope:** PHASES.md P2 deliverables: the D2 tripwire (targeted-signal regression past the KPI band, else ≥25% relative with ≥3 post-ship occurrences, OR ≥2 attributable fresh incidents) and D3 attribution (timestamp ∈ window AND surface ∈ `canary.surfaces`; unknown surfaces never attribute; a shared surface counts against all matching open canaries; clustered incidents preferred, raw deny/breadcrumb fallback).
- **TDD:** yes
- **Files to create/modify:** `user/scripts/efficacy-eval.py`
- **Test files:** `user/scripts/test_efficacy_eval.py`
- **Test expectations:**
  - Trip on a fixture telemetry ledger regressing the targeted signal past 25% within the window (band numbers reported).
  - Trip on 2 fixture incidents whose emitting surface ∈ `canary.surfaces`.
  - NO trip on 2 fixture incidents on an UNRELATED surface; those entries are listed as unattributed but not counted.
  - An incident with no resolvable surface never attributes.
  - A surface shared by two open canaries counts the incident against BOTH.
- **Implementation goal:** Compute the regression band against the frozen baseline (reuse the evaluator's existing verdict math where applicable) and the incident count via the incident-scan readers — `read_hook_events`, `read_legacy_crumbs`, `collect_clusters`, and the `_first_token` deny→surface signature [VERIFY: grep -n "def read_hook_events\|def read_legacy_crumbs\|def collect_clusters\|def _first_token" user/scripts/incident-scan.py] — plus `lazy_core.read_deny_ledger` [VERIFY: grep -n "read_deny_ledger" user/scripts/incident-scan.py]. Prefer clustered incidents; fall back to raw deny/breadcrumb. Map each incident's emitting surface to a repo-relative path and match against `canary.surfaces`.
- **Spec requirements:** SPEC D2 tripwire semantics; D3 Resolution (surface-based, unknown-never-attributes, shared-counts-against-all); Technical Design "Incident sources (v1)".
