### Phase: harness-change-canary-rollback P4 — Steady-state handoff + surfacing + KPI

**Goal:** Windows close with a status stamp + a `## Canary <date>` record section; the retro cites canary outcomes alongside efficacy verdicts; the canary's own trip-precision KPI is registered and computed. Monitoring drops back to normal cadence.
**Entry criteria:** Phase 3 landed (`tripped` / enqueue stamps exist; trip-precision counts closes-as-noise against trips).
**SPEC.md references:** D7 (steady-state handoff), KPI Declaration (`canary-trip-precision`); Technical Design "no trip + window matured → closed-clean [handoff]"; Validation Criteria "Clean close + handoff".

#### WU-8 — Window-close stamps + `## Canary <date>` record section

- **Scope:** PHASES.md P4 deliverables: on maturity with no trip stamp `canary.status: closed-clean` (or `closed-clean (no-data)`); a tripped canary keeps `tripped` + the enqueued id; append a `## Canary <date>` section to the record body (runs observed, signal movement, incidents attributed none/list). Handoff invariants: a clean canary does not pre-judge the efficacy verdict; a tripped one does not skip it; the watcher stops waking a closed/tripped record.
- **TDD:** yes
- **Files to create/modify:** `user/scripts/efficacy-eval.py`
- **Test files:** `user/scripts/test_efficacy_eval.py`
- **Test expectations:**
  - A matured-no-trip fixture stamps `canary.status: closed-clean` and appends a `## Canary <date>` section summarizing the window.
  - A matured no-observable-runs fixture stamps `closed-clean (no-data)`.
  - After close, the record is NOT re-enumerated by a subsequent `--canary` run (watcher stops waking it).
  - The efficacy review path is unaffected — a `closed-clean` canary does not write or alter any efficacy verdict field.
- **Implementation goal:** Add the close-stamp + `## Canary` section writer to the `--canary` mode, reusing `_write_record` [VERIFY: grep -n "def _write_record" user/scripts/efficacy-eval.py] and the body-append pattern the efficacy review already uses for its `## Review <date>` sections [VERIFY: grep -n "## Review" user/scripts/efficacy-eval.py]. Keep canary writes scoped to `canary.*` + the appended section — never touch efficacy verdict fields.
- **Spec requirements:** SPEC D7 Resolution; Technical Design "[handoff]"; User Experience "Window close, no trip".
- **Batch:** 1 (file-disjoint from WU-9 + WU-10 → may run in parallel)
