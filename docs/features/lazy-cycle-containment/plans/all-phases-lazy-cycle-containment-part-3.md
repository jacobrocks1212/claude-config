### Phase: lazy-cycle-containment P4 — PreToolUse containment hook (C2)

**Goal:** Add `user/hooks/lazy-cycle-containment.sh` + `settings.json` wiring so that, while the cycle marker is present, the hook DENIES in-flight the tool calls a runaway needs; fast-path-exits when the marker is absent; fails OPEN on hook error (the C3 refusal is the backstop).

**Entry criteria:** part-2 Phases 2 (marker file + `commit_tally`) and 3 (C3 refusal backstop) complete.

**SPEC.md references:** Technical Design §C2 (PreToolUse containment hook); Validation Criteria rows "Hook denies the next-route probe", "Hook denies lifecycle/runtime commands", "Hook denies a 2nd-feature commit", "Hook is inert without marker", "new-hook entry surfaces restart warning".

#### WU-1 — hook script: fast-path, loop-formation/lifecycle/recursive-dispatch deny, fail-OPEN

**Scope:** PHASES.md P4 deliverables: fast-path `test -f` exit; loop-formation deny; runtime-lifecycle deny; recursive-dispatch deny; fail-OPEN on hook error.

**TDD:** yes (WU-3 holds the bash test harness; author the hook to satisfy it).

**Files to create/modify:**
- `user/hooks/lazy-cycle-containment.sh` — NEW.

**Implementation goal:**
- Model the hook on the existing PreToolUse hooks `user/hooks/lazy-route-inject.sh` and `user/hooks/lazy-dispatch-guard.sh` [VERIFY: ls user/hooks/lazy-route-inject.sh user/hooks/lazy-dispatch-guard.sh] — same stdin-JSON parsing, same `permissionDecision` output shape, same fail-OPEN-on-error breadcrumb pattern.
- Fast-path: `test -f "${LAZY_STATE_DIR:-$HOME/.claude/state}/lazy-cycle-active.json"`; if absent, ALLOW (emit nothing / the allow decision) and exit immediately — interactive sessions and orchestrator-between-cycles untouched.
- While the marker is present, parse the Bash command from the PreToolUse payload and DENY (`permissionDecision: deny` + a corrective `permissionDecisionReason`) when it matches:
  - **Loop-formation:** `lazy-state.py`/`bug-state.py` invoked with any of `--probe`, `--emit-prompt`, `--repeat-count`, `--repeat-count-peek`, `--run-start`, `--run-end`, `--apply-pseudo`, `--enqueue-adhoc`, `--emit-dispatch`. ALLOW the narrow ops `--neutralize-sentinel` and `--verify-ledger`.
  - **Runtime-lifecycle:** `npm run dev:kill`, `npm run dev:restart`, `dev:kill`, `dev:restart`, `kill-port 3333`, `kill-port 1420`.
  - **Recursive dispatch:** an `Agent` tool call while the marker is present (matched on the PreToolUse `tool_name == "Agent"`).
- The corrective `permissionDecisionReason` should read: "you are a single cycle subagent — STOP after your commit+push+report; routing the next cycle is the orchestrator's job" and align with the Phase-6 cycle-prompt terminal-stop wording (part-4).
- Fail-OPEN: any hook error (malformed JSON, missing tool, unexpected state) logs a breadcrumb and ALLOWS — a broken hook must never wedge the pipeline (the C3 refusal is the backstop). Mirror `lazy-route-inject.sh`'s breadcrumb path.

**Spec requirements:** SPEC §C2 — the full deny-set, allow-list, and "Fail-OPEN on hook error (log a breadcrumb), consistent with the existing route-inject hook".

**Batch:** 1

#### WU-2 — second-feature commit tripwire + commit-count backstop

**Scope:** PHASES.md P4 deliverables: second-feature commit tripwire (with shared-root carve-outs); commit-count backstop reading/incrementing `commit_tally`.

**TDD:** yes (WU-3 harness covers it).

**Files to create/modify:**
- `user/hooks/lazy-cycle-containment.sh` — extend WU-1's hook with the `git commit` handling.

**Implementation goal:**
- On a `git commit` command while the marker is present: resolve the staged feature dir(s) from `git diff --cached --name-only`. DENY if ANY staged path is under a DIFFERENT feature dir than the marker's `feature_id` (read from `lazy-cycle-active.json`). Carve-outs (always allowed even when cross-feature): `docs/features/queue.json`, `docs/features/ROADMAP.md`, repo-root `CLAUDE.md`, and the feature's own dir (`docs/features/<feature_id>/`).
- Commit-count backstop: read `commit_tally` from the marker; DENY a `git commit` beyond a generous absolute ceiling (default 25/dispatch). On each ALLOWED `git commit`, increment `commit_tally` in the marker file (read-modify-write the JSON; keep it minimal — `jq` if available, else a small inline edit consistent with how other hooks mutate state).
- Both denies emit a corrective `permissionDecisionReason` consistent with WU-1.

**Spec requirements:** SPEC §C2 — "Second-feature commit tripwire ... (with carve-outs for shared roots ...)" and "Commit-count backstop: a generous absolute ceiling (default 25 commits/dispatch ...) read from the marker's `commit_tally`; DENY beyond it. The tally is incremented by the hook on each allowed `git commit`."

**Batch:** 1

#### WU-3 — settings.json wiring + bash hook-test harness

**Scope:** PHASES.md P4 deliverables: `settings.json` PreToolUse wiring (Bash + Agent); tests driving the hook with crafted Bash payloads.

**TDD:** yes (author the harness FIRST as the failing spec for WU-1/WU-2).

**Files to create/modify:**

**Test expectations (driven against crafted PreToolUse JSON payloads + a tmp `LAZY_STATE_DIR`):**
- deny next-route probe: marker present + `lazy-state.py --probe` → `permissionDecision: deny`.
- deny lifecycle: marker present + `lazy-state.py --run-end` and + `npm run dev:kill` → deny.
- deny 2nd-feature commit: marker present (`feature_id: A`) + a `git commit` whose `git diff --cached --name-only` shows `docs/features/B/...` → deny; a commit staging only `docs/features/A/...` (or a carve-out path) → ALLOW (use a fixture for the staged-path output).
- allow allow-listed ops: marker present + `lazy-state.py --neutralize-sentinel` / `--verify-ledger` → ALLOW.
- recursive-dispatch deny: marker present + an `Agent` tool payload → deny.
- fast-path allow: marker ABSENT + any Bash payload → ALLOW (no deny).
- fail-OPEN: malformed JSON payload → ALLOW + a breadcrumb (no wedge).
- commit-count backstop: marker present with `commit_tally` at the ceiling + `git commit` → deny.

**Implementation goal:** Harness passes against WU-1/WU-2.
