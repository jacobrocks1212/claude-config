**Goal:** Detect in-harness self-edit runs via a symlink-resolves-under-git-toplevel predicate, surface it on the probe JSON as `self_edit_mode`, and install the orchestrator's governing-file reload discipline + new-hook-registration restart surfacing so this spec's later governing-prose edits take effect on the running orchestrator.

**Entry criteria:** None (first phase, per SPEC ordering rationale).

**SPEC.md references:** Technical Design §C8 (Self-edit reload discipline); Implementation Phases §Phase 1; Validation Criteria rows "self_edit_mode true only in-harness", "governing-prose edit triggers re-read", "auto-refreshing surfaces NOT flagged", "new-hook entry surfaces restart warning".

#### WU-1 — self-edit predicate + probe `self_edit_mode` enrichment

**Scope:** PHASES.md P1 deliverables: the `lazy_core.py` self-edit predicate; `--probe` / `emit_cycle_prompt` carrying `self_edit_mode` (and optionally `governing_files_touched`); auto-refresh-boundary documentation in `lazy_core.py`.

**TDD:** yes (WU-3 holds the tests; this WU implements to satisfy them — author the predicate and probe wiring against the WU-3 test expectations).

**Files to create/modify:**
- `user/scripts/lazy_core.py` — add `self_edit_mode()` predicate + probe-JSON enrichment + auto-refresh-boundary comment block.
- `user/scripts/lazy-state.py` — ensure the probe CLI surfaces `self_edit_mode` (thin wrapper over `lazy_core`).

**Implementation goal:**
- Add a `self_edit_mode(repo_root)` predicate to `lazy_core.py` that returns `True` iff `~/.claude/skills`, `~/.claude/scripts`, AND `~/.claude/hooks` ALL resolve (after `os.path.realpath` symlink resolution) to a path UNDER the run's `git rev-parse --show-toplevel`. Semantically-correct (robust to the repo cloned elsewhere); NOT a cwd-basename match. Resolve `~` via `os.path.expanduser`. If any of the three paths is missing or resolves outside the toplevel, return `False`. If `git rev-parse --show-toplevel` fails (not a git repo), return `False`.
- Surface the flag on the probe output the orchestrator reads. The existing probe/prompt emission lives in `emit_cycle_prompt` [VERIFY: grep -n "def emit_cycle_prompt" user/scripts/lazy_core.py] — add `self_edit_mode: true|false` to the emitted JSON/context, and optionally a `governing_files_touched` list derived from the last commit's `git diff --name-only HEAD~1 HEAD` intersected with the governing-file set (see WU-2 for the canonical set).
- Add a documented auto-refresh-boundary comment block in `lazy_core.py` listing the surfaces that MUST NOT be flagged for reload (they auto-refresh): `lazy_core.py`/`lazy-state.py` (fresh subprocess per probe), `cycle-base-prompt.md`/addenda/`loop-block.md` (re-read by `emit_cycle_prompt` every probe [VERIFY: grep -n "cycle-base-prompt" user/scripts/lazy_core.py]), hook `.sh` bodies (read per invocation), downstream skill prose (loaded fresh per dispatch).

**Spec requirements:** SPEC §C8 — "Self-edit predicate ... returns true when `~/.claude/skills`, `~/.claude/scripts`, AND `~/.claude/hooks` all resolve under the run's `git rev-parse --show-toplevel` ... This is the semantically-correct predicate (robust to the repo being cloned elsewhere); NOT a brittle cwd-basename match." and "Probe enrichment: `emit_cycle_prompt`/`--probe` carry the `self_edit_mode` flag (and optionally a `governing_files_touched` list ...)".
