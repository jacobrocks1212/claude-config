## Plan-specific execution notes

- **INLINE execution.** These WUs are a small mechanical/logic batch in one repo — the executing session performs the work inline (Read/Edit/Write), test-first per WU. Sub-subagent dispatch is optional, not required.
- **Fail-open is load-bearing and must be tested, not just asserted.** Every error path in the hook (unreadable JSON, missing `agent_id`, unresolvable repo, breadcrumb I/O failure, no python) MUST exit 0 (allow). A backstop hook that can itself wedge the pipeline is worse than the wedge it prevents. WU-1's tests are the proof.
- **Breadcrumb path is OUTSIDE any repo** — `<claude-state>/subagent-stops/<agent_id>.json`, where `<claude-state>` is `lazy_core.claude_state_dir()` (honors `LAZY_STATE_DIR`, which is how the tests point it at a temp dir). Never write the breadcrumb into a repo working tree (it would dirty the tree the predicate inspects).
- **Loop-guard depends ONLY on the documented `agent_id`** — NEVER on the undocumented `stop_hook_active` (confirmed absent from the `SubagentStop` input schema; see SPEC "Platform confirmation"). Cite that confirmation in a header comment of the hook.
- **Active-plan resolution + fail-open bias.** Predicate condition 2 ("active plan status != `Complete`") gates the block. If no active plan resolves for the repo, or its status/checkboxes cannot be read, the hook ALLOWS (exit 0) — consistent with the SPEC's explicit "bias to false-negative (let it stop) over false-positive." Condition 3's git-dirty signal is checked via `git -C <repo> status --porcelain`.

### Phase: subagent-wedge-backstop-hook P1 — SubagentStop wedge-backstop hook + loop-guard + registration + tests

**Goal:** A fail-open `SubagentStop` hook that blocks a genuinely-wedged dispatched subagent at most once (breadcrumb keyed on `agent_id`), forcing commit+complete or a `BLOCKED.md` instead of a dead stop.
**Entry criteria:** none (SPEC `**Depends on:** (none)`; working tree clean).
**SPEC.md references:** "Design (the authorized option)" (predicate / loop-guard / block reason / fail-open / breadcrumb lifecycle); "Platform confirmation"; "Testability".

#### WU-1 — `subagent-wedge-backstop.sh` hook + hermetic tests

- **Scope (PHASES deliverables 1–5, 7, 8):**
  - `user/hooks/subagent-wedge-backstop.sh` — `SubagentStop` hook reading input JSON (`agent_id`, `cwd`, `session_id`); resolves the repo from `cwd`.
  - **Predicate:** block only when (run marker present for the repo) AND (active plan status != `Complete`) AND (git working tree dirty OR active plan has unchecked WU checkboxes). Otherwise exit 0 (allow).
  - **Loop-guard:** self-managed breadcrumb at `<claude-state>/subagent-stops/<agent_id>.json` (absolute path outside any repo); block at most ONCE per `agent_id` (present breadcrumb ⇒ allow). NO dependency on `stop_hook_active`.
  - **Block action:** exit 2 with an actionable `reason` ("You are stopping with pending plan work and the completion protocol has not run. Commit your work and complete the plan, or write BLOCKED.md with the obstacle, then stop."); write the breadcrumb BEFORE blocking.
  - **Fail-open:** any error / missing field / unresolvable repo / breadcrumb I/O failure / no python ⇒ exit 0 (allow). Bias to false-negative.
  - **Breadcrumb lifecycle:** staleness sweep on entry (GC breadcrumbs older than a threshold) AND a `SessionEnd`-mode branch (when invoked with SessionEnd input — `session_id` present, `agent_id` absent — GC breadcrumbs recorded under that `session_id`; store `session_id` + write time in each breadcrumb). GC failure is non-fatal.
- **TDD:** yes.
- **Files to create/modify:**
  - `user/hooks/subagent-wedge-backstop.sh` (create)
  - `user/scripts/test_hooks.py` (modify — append new cases)
- **Test files:** `user/scripts/test_hooks.py`
- **Test expectations** (author FIRST; watch fail against the absent hook, then implement):
  - predicate-true (marker present + non-Complete plan + dirty tree) → **block once** (exit 2, reason present, breadcrumb written).
