### Standard seam-move procedure (referenced by every WU below)

1. **Locate:** re-verify the seam's line ranges in `user/scripts/lazy_core/_monolith.py` by section-header grep (the RESEARCH_SUMMARY section map is the guide; line numbers are approximate — the pattern is authoritative). Record the exact ranges.
2. **Slice:** scratchpad script reads `_monolith.py`, extracts the ranges VERBATIM into `user/scripts/lazy_core/<seam>.py` (adding only: module docstring, `from __future__ import annotations` if the monolith has it, the module's own stdlib imports, `from ._ctx import ...` for kernel names, and `from . import <other-seam>` / `from .<mod> import <name>` per note 6), and deletes the ranges from `_monolith.py`, inserting the minimal import-back line (`from .<seam> import <names still referenced by remaining monolith code>` for un-patched names; `from . import <seam>` + attribute-access rewrite for patched names still referenced — enumerate these in the WU report).
3. **Facade map:** the script AST-parses the new seam module's top-level `def`/`class`/assignment names and appends `_SUBMODULE_BY_NAME` entries (`<name>: "<seam>"`) to `__init__.py`; adds the seam to `_ALL_SUBMODULES`. Collision check: a name mapped twice = hard error.
4. **Test redirect:** scratchpad script re-points `lazy_core._monolith.<name>` → `lazy_core.<seam>.<name>` in `test_lazy_core.py` for every name whose CONSUMER-under-test moved (start from the moved-name list ∩ patched-name census; run the suite — a failing patch site names the next redirect). Receipt: per-name counts.
5. **Compile + suite:** `python -c "import sys; sys.path.insert(0,'user/scripts'); import lazy_core; lazy_core.load_all()"` then the full battery. Collected count must remain **1135** with an identical name set (`python -m pytest user/scripts/test_lazy_core.py --collect-only -q` diff against the pre-phase capture).
6. **Commit** (message: `refactor(lazy-core): Phase 2 — extract <seam>.py (move-only)`), then the importer-diff guard (note 4).
