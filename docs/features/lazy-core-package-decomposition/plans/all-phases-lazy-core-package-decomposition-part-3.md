#### WU-1 — Split mapping + `conftest.py` + guard-generalization design

- **Scope:** PHASES P3 bullet (first half): the `tests/` per-seam layout + `conftest.py` `tmp_repo` fixture; plus the split map the WU-2 splitter executes.
- **TDD:** no (net-new scaffolding; the receipt in note 3 is the gate).
- **Files to create/modify:** `user/scripts/tests/test_lazy_core/__init__.py` (empty), `user/scripts/tests/test_lazy_core/conftest.py` (new), scratchpad: `split-map.json` (not committed).
- **Implementation goal:**
  1. `conftest.py`: (a) `sys.path.insert(0, str(Path(__file__).resolve().parents[2]))` so `import lazy_core` resolves from any collection root (mirrors the state scripts' insertion); (b) the `tmp_repo` pytest fixture — tmp dir + minimal `docs/features` skeleton, the pattern hand-rolled 726 times today (define it now; ADOPTION IS INCREMENTAL per SPEC D5 — this phase does NOT rewrite existing tests onto it; new tests may use it).
  2. **Split map (scripted analysis, scratchpad):** AST-parse `test_lazy_core.py`; assign every top-level statement block (test functions, helper functions/classes, `_TESTS` extension blocks, section comments, module constants) to a target file, preserving source order within each: `test_docmodel.py`, `test_depdag.py`, `test_hostcaps.py`, `test_notifyplane.py`, `test_statedir.py`, `test_gates.py`, `test_dispatch.py`, `test_runtimeplane.py`, `test_markers.py`, `test_pseudo.py`, `test_ledgers.py`, `test_misc.py` (residue: cross-cutting helpers, the module header, guards). Assignment heuristic: the seam whose functions the test predominantly calls (grep the test body for `lazy_core.<name>` / `lazy_core.<seam>.<name>` tokens against the facade map); ambiguous/cross-seam tests → `test_misc.py`. Rules baked into the map: (a) a helper used by tests in N files goes to `conftest.py` (as a plain function) or is duplicated ONLY if trivial — prefer conftest; (b) each meta-guard travels WITH its audited population, and its `Path(__file__)` source read is GENERALIZED to iterate every `*.py` in the split dir (a superset scan is always safe); (c) each `_TESTS` extension block is split so each target file carries `_TESTS` entries exactly for the tests it contains, plus a small per-file `main()` (verbatim copy of the existing runner shape) so `python3 user/scripts/tests/test_lazy_core/test_<seam>.py` still works; (d) the module docstring/header lands in `test_misc.py` (or a shared `_util.py`).
  3. Emit the map as JSON (function-name → target file) + a coverage assertion: every line range of the source file is assigned exactly once.
- **Spec requirements:** SPEC D5.
