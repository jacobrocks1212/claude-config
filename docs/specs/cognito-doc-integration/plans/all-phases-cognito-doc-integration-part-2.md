#### WU-4.1 — `lazy_coord.py` coordination module + `--test`  *(TDD: yes)*

- **Scope (PHASES deliverables):** the net-new coordination module exporting `acquire_lock`, `release_lock`, `acquire_lease`, `heartbeat`, `verify_fencing`, `reclaim_expired`, `release_lease`; the `os.mkdir` global lock with backoff; the `leases.json` schema; worktree-pool provisioning; the deterministic scrub-to-clean sequence; `index.lock` backoff; and `--test` fixtures for the five concurrency cases.
- **Files to create/modify:**
  - `C:\Users\JacobMadsen\source\repos\claude-config\user\scripts\lazy_coord.py` (net-new — SEPARATE from `lazy_core.py`)
- **Test files / expectations (`--test`, offline, `tempfile` trees):**
  1. **scoped run proxy:** (covered in WU-4.2/4.3 baseline tests; here ensure lease ops don't perturb queue).
  2. **leased run:** seed a lease for `wi_id`; a second `acquire_lease` for the same `wi_id` cannot claim it (no double-claim).
  3. **reclamation:** seed an expired lease (`heartbeat_timestamp + ttl_seconds < now`); `reclaim_expired` removes it and frees/scrubs its slot.
  4. **fencing:** seed a lease with `term_token=5`; `verify_fencing(expected_token=4)` raises.
  5. **mkdir mutual exclusion:** simulate two acquire attempts; exactly one succeeds, the other gets `FileExistsError` and retries.
  All offline; do NOT require a live cognito clone for the lock/lease/fencing tests (mock the pool dir). Time is injected (pass a `now` param or monkeypatch) so the reclamation test is deterministic — do NOT call wall-clock directly in a way that makes the test flaky.
- **Implementation goal:**
  - Header `#!/usr/bin/env python3` + docstring + `from __future__ import annotations`; stdlib only (`os`, `json`, `time`, `subprocess`, `pathlib`, `errno`).
  - `acquire_lock(lock_dir, timeout)`: `os.mkdir(lock_dir)`; on `FileExistsError`, sleep with exponential backoff and retry until `timeout`, then raise. `release_lock(lock_dir)`: `os.rmdir`.
  - Lease ops mutate `leases.json` only via atomic temp-file `os.replace` AFTER acquiring the global lock. `acquire_lease(leases_path, wi_id, worker_pid, slot, ttl)`: reclaim-expired-first, claim a free slot, increment `term_token` (fencing), write `{worker_pid, worktree_slot, term_token, heartbeat_timestamp, ttl_seconds}`. `heartbeat(leases_path, wi_id, expected_token)`: re-assert lock, verify token, refresh timestamp. `verify_fencing(leases_path, wi_id, expected_token)`: raise on mismatch. `reclaim_expired(leases_path, pool_dir, ttl)`: drop expired leases + scrub their slots. `release_lease(leases_path, wi_id, expected_token)`: fencing-checked drop.
  - `leases.json` schema EXACT: `{ "<wi_id>": { "worker_pid": int, "worktree_slot": str, "term_token": int, "heartbeat_timestamp": str, "ttl_seconds": int } }`.
  - Worktree pool: `provision_pool(cognito_root, pool_dir, K)` → `git -C <cognito_root> worktree add <pool_dir>/wt-NN` for N in 0..K-1; apply git concurrency config to the cognito repo: `git config gc.auto 0`, `git config core.filemode false`, `git config core.autocrlf input`. Serialize all `git fetch`/`git push` under the global lock.
  - Scrub-to-clean (ordered, on slot reuse): (1) `rm -f .git/worktrees/<slot>/index.lock`; (2) under global lock `git fetch origin`; (3) `git checkout --detach origin/main`; (4) `git reset --hard origin/main`; (5) `git clean -fdx`; (6) `git checkout -b p/<wi_id>-<slug>`. NO submodule step (`.gitmodules` absent; reinstate if ever added). `index.lock` contention → exponential backoff with per-retry logging.
- **Spec requirements:** SPEC Layer 4 in full; Locked Decisions 16–18; Validation Criteria rows "No double-claim", "Fencing prevents zombie writes", "mkdir-lock mutual exclusion", "Worktree isolation + scrub", "gc disabled".
