## Phase: cognito-doc-integration P3 — WI→Doc Materialization

**Goal:** Discrete, idempotent materialization: auto-route a WI by type, thin-copy verbatim into doc format, enqueue, stamp the AB# link, record in `materialized.json`, and install the `STALE_UPSTREAM.md` divergence gate. Adds a new bug `enqueue_adhoc` to `bug-state.py` and sentinel/materialized helpers to `lazy_core.py`.
**Entry criteria:** P1 complete (mirror with `type`/`changedDate`/`acceptanceCriteria`/`description`/`url`); P0 config (type→pipeline map) readable.
**SPEC.md references:** Layer 3 — Materialization + orchestration; Locked Decision 20 (STALE_UPSTREAM); type→pipeline map.

### Key design decisions (grounded — apply verbatim)
- **`--materialize-wi <id>` is added to `lazy-state.py`** (audit confirmed backward-compatible: early return in `main()` before `compute_state()`; `compute_state` gets an optional param later). NO separate `materialize-wi.py`.
- **Cross-script enqueue:** `lazy-state.py` and `bug-state.py` are hyphenated → NOT importable as modules. For a **bug-type** WI, the materialize path invokes `bug-state.py` via **subprocess** (`python bug-state.py --enqueue-adhoc ...`). Therefore WU-3.2 must add both the `enqueue_adhoc` function AND a `--enqueue-adhoc` CLI flag to `bug-state.py` (mirroring lazy-state's `--enqueue-adhoc` wiring at ~line 2474). For a **feature-type** WI, materialize calls the in-process `enqueue_adhoc()` at `lazy-state.py:214` verbatim.
- **STALE_UPSTREAM write stays OUT of `compute_state()`** — `compute_state` sentinel handling is READ-ONLY (verified). The stale-check that *writes* `STALE_UPSTREAM.md` runs in the materialize/probe path. `compute_state` only *reads* it (via the new `lazy_core.read_stale_upstream`) and returns a halt state.
