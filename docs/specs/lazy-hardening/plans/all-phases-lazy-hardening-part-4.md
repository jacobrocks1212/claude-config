## Phase: lazy-hardening P4 — Parked-decision protocol + notifications (D1/D2)

**Goal:** Opt-in park-and-continue (`--park`) with batched flush; two-key auto-accept (`--park` only); push on every park/halt/flush/run-end in both modes. Default (no flag) = byte-for-byte existing halt-and-wait.
**Entry criteria:** Phase 3 complete; tree clean.

### Work Units

#### WU-1 — Script `--park-needs-input` mode + `parked[]` output  *(TDD: yes)*
- **Scope:** "Script `--park-needs-input` mode + `parked[]` output array, passed only when skill got `--park` (wrappers keep halt behavior; probe output unchanged without the flag)"
- **Files:** `user/scripts/lazy-state.py` + `user/scripts/bug-state.py` (CLI flag) + shared impl in `user/scripts/lazy_core.py` [VERIFY: `grep -n "add_argument(\"--" user/scripts/lazy-state.py | head`]
- **Implementation goal:** add `--park-needs-input`: a feature/bug carrying an unresolved `NEEDS_INPUT.md` is **skipped (not halted)** and reported in a new `parked[]` output array (`id`, sentinel path, decision count, parked-since). **Without the flag, probe output is byte-unchanged.** Parked items re-enter automatically once the sentinel is resolved/renamed (existing routing).
- **Test:** WITHOUT flag → NEEDS_INPUT fixture still emits `needs-input` halt, probe byte-identical to Phase-1 baseline. WITH flag → `parked[]` populated, item skipped not halted, resolved sentinel re-enters.
- **Batch:** 1

#### WU-2 — `--park` skill flag parsing + banner/report  *(TDD: no — prose)*
- **Scope:** "`--park` skill invocation flag parsed in all 3 batch orchestrators; recorded in start banner + final report; no flag → existing Step 1g halt behavior unchanged"
- **Files:** the 3 batch SKILLs [VERIFY: `grep -n "max_cycles" user/skills/lazy-batch/SKILL.md | head`] (parse `--park` alongside `max_cycles`; record in start banner + final report)
- **Batch:** 1 (skills — no overlap with WU-1 scripts)

#### WU-3 — PushNotification at every park / halt / flush / run-end (BOTH modes)  *(TDD: no — prose)*
- **Scope:** "PushNotification at every park / halt / flush / run end (both modes)"
- **Files:** the 3 batch SKILLs (document orchestrator-fired PushNotification with running parked-count) [VERIFY: `grep -n "PushNotification" user/skills/lazy-batch/SKILL.md`]
- **Batch:** 2 (overlaps WU-2 on batch SKILLs)

#### WU-4 — D1 flush protocol (`--park` only)  *(TDD: no — prose)*
- **Scope:** "D1 flush protocol (`--park` only): batched AskUserQuestion (≤4/call, Zero-Context Briefing preserved) at first opportunity (operator message / out of unparked work / run end); decision-apply per answer"
- **Files:** `lazy-batch` + `lazy-bug-batch` + `lazy-batch-cloud` SKILLs
- **Implementation goal (D1 locked):** flush all parked decisions as batched AskUserQuestion(s) — ≤4 questions/call, preserving the Zero-Context Operator Briefing (verbatim context re-print in chat BEFORE the call, options 1:1 with chat) — at first opportunity: (a) any operator message mid-run, (b) no unparked work remains, (c) run end. After answers: existing decision-apply machinery per item; sentinels renamed; loop continues.
- **Batch:** 3 (overlaps WU-3 on batch SKILLs)

#### WU-5 — D2 two-key auto-accept (`--park` only)  *(TDD: no — prose; NEEDS_INPUT schema is the only mechanical change)*
- **Scope:** "D2 two-key auto-accept (`--park` only): `class: mechanical` + input-audit concurrence → recommended option auto-accepted, `resolved_by: auto-two-key`, receipt log + run-end digest; any disagreement → product → park; no auto-accept ever without `--park`"
- **Files:** `~/.claude/skills/_components/sentinel-frontmatter.md` (extend NEEDS_INPUT schema with `class: mechanical|product` authored by the cycle subagent + `audit_concurs: true` from Step 1d.5) + the 3 batch SKILLs (auto-accept logic + run-end digest table)
- **Implementation goal (D2 locked):** auto-accept only when BOTH keys agree `mechanical` AND a recommended option exists → take it, append resolution `resolved_by: auto-two-key`, log in receipt + a run-end "auto-accepted decisions" digest table with links. Any disagreement/absence/missing audit → `product` → park. **Never auto-accept without `--park`.**
- **Batch:** 4 (overlaps WU-4 on batch SKILLs; touches sentinel-frontmatter)

#### WU-6 — Cache-boundary note  *(TDD: no — docs)*
- **Scope:** "Cache-boundary note documented in both batch skills"
- **Files:** `lazy-batch` + `lazy-batch-cloud` SKILLs (note that flushes (b)/(c) are also the cache-rebuild boundaries; do not interleave unrelated long waits between park and flush)
- **Batch:** 5 (overlaps WU-5 on batch SKILLs)
