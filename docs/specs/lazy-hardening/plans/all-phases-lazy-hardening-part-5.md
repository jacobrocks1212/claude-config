## Phase: lazy-hardening P5 — Script-ification of the orchestrator loop

**Goal:** Cut per-cycle orchestrator overhead from ~6 messages to ≤3 by moving deterministic mechanics into the scripts (`lazy-state.py` + `bug-state.py`, shared impl in `lazy_core.py`).
**Entry criteria:** Phases 1–2 complete; tree clean.

### Work Units

#### WU-1 — `--verify-ledger <spec_path>`  *(TDD: yes)*
- **Scope:** "`--verify-ledger <spec_path>` (replaces 5 prose blocks)"
- **Files:** shared impl in `user/scripts/lazy_core.py` + CLI in `user/scripts/lazy-state.py` + `user/scripts/bug-state.py` [VERIFY: `grep -n "def count_deliverables\|def has_completion_receipt" user/scripts/lazy_core.py` → 784,192]
- **Implementation goal:** `--verify-ledger` returns pass/fail + the failing check across: clean tree, HEAD==origin, plan `status: Complete`, zero non-verification `- [ ]`. Replaces 5 duplicated prose blocks (lazy Step 4, lazy-bug Step 4, both batch 1e.4a blocks, cloud 1e) — those are deleted in WU-6.
- **Test:** one fixture per failing check (dirty tree, behind origin, plan not Complete, unchecked non-verification deliverable) → correct fail+reason; all-green → pass.
- **Batch:** 1

#### WU-2 — `--apply-pseudo <name> <spec_path>` + receipt-write ownership resolution  *(TDD: yes)*
- **Scope:** "`--apply-pseudo <name> <spec_path>` for all deterministic sentinel/receipt writes; receipt-write ownership contradiction resolved (script is single author)"
- **Files:** `user/scripts/lazy_core.py` (reuse `write_completed_receipt` [VERIFY: `grep -n "def write_completed_receipt" user/scripts/lazy_core.py` → 208; currently wired only to `--backfill-receipts`]) + `user/scripts/lazy-state.py` + `user/scripts/bug-state.py` + `~/.claude/skills/_components/completion-integrity-gate.md`
- **Implementation goal:** script the deterministic writes: `__write_validated_from_results__`, `__write_validated_from_skip__`, `__write_deferred_non_cloud__`, `__flip_plan_complete_cloud_saturated__`, and the receipt+status-flip+sentinel-cleanup core of `__mark_complete__`/`__mark_fixed__` [VERIFY: `grep -n "__write_validated_from_results__\|__write_validated_from_skip__\|__write_deferred_non_cloud__\|__flip_plan_complete_cloud_saturated__" user/scripts/lazy-state.py`]. Gate CHECKS stay put; the WRITES become mechanical. Resolve the contradiction (completion-integrity-gate.md says the gate writes the receipt; consumers say they do) by making `--apply-pseudo` the SINGLE author + updating both texts.
- **Test:** apply-pseudo idempotency (re-run = no-op/stable) + refusal when gate inputs absent.
- **Batch:** 2 (overlaps WU-1 on lazy_core/lazy-state/bug-state)

#### WU-3 — `--neutralize-sentinel <path>` with collision handling  *(TDD: yes)*
- **Scope:** "`--neutralize-sentinel <path>` with collision handling"
- **Files:** `user/scripts/lazy_core.py` + CLI in `user/scripts/lazy-state.py`/`bug-state.py`
- **Implementation goal:** scripted rename to the canonical `*_RESOLVED_<date>` form with collision handling (the rename collided once in practice — handle the existing-target case deterministically).
- **Test:** neutralize collision case (target already exists) → deterministic non-clobbering rename.
- **Batch:** 3 (overlaps WU-1/WU-2 on the scripts)

#### WU-4 — Persisted probe signature → `repeat_count`  *(TDD: yes)*
- **Scope:** "Persisted probe signature → `repeat_count` in output"
- **Files:** `user/scripts/lazy-state.py` + `user/scripts/bug-state.py` (+ shared in `lazy_core.py`)
- **Implementation goal:** the script stores the last emitted `(id, sub_skill, args, step)` tuple (e.g. `.lazy-state-last.json` under the repo's logs/docs dir) and emits `repeat_count` so loop detection is mechanical, not prose.
- **Test:** `repeat_count` increments across identical probes; resets on signature change.
- **Batch:** 4 (overlaps WU-2/WU-3 on the scripts)

#### WU-5 — Single probe payload (git guards + cycle header)  *(TDD: yes)*
- **Scope:** "Probe payload includes git-guard results + pre-formatted cycle header"
- **Files:** `user/scripts/lazy-state.py` + `user/scripts/bug-state.py` (+ shared in `lazy_core.py`)
- **Implementation goal:** fold git-guard results (tree clean? HEAD==origin? unpushed?) and a pre-formatted cycle-header block into the probe JSON, so the orchestrator's happy-path turn becomes: read payload → dispatch → record.
- **Test:** probe JSON contains git-guard fields + cycle-header block; matches expected shape.
- **Batch:** 5 (overlaps WU-4 on the scripts)

#### WU-6 — Skill text consumes subcommands; delete superseded prose  *(TDD: no — prose; SEPARATE commit)*

Commit SEPARATELY from the script WUs.
