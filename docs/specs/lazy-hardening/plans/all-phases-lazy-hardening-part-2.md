#### WU-1 — Receipt content validation in `lazy_core.has_completion_receipt()`  *(TDD: yes)*
- **Scope:** "`lazy_core.has_completion_receipt()` validates frontmatter (`kind:`, `provenance:`); malformed → treated missing + diagnostic"
- **Files:** `user/scripts/lazy_core.py` [VERIFY: `grep -n "def has_completion_receipt" user/scripts/lazy_core.py` → 192]; tests in `user/scripts/test_lazy_core.py`
- **Implementation goal:** the current bare `.exists()` (a `touch COMPLETED.md` defeats the completion-unverified gate) must instead require parseable frontmatter with `kind: completed|fixed` AND a `provenance:` value. - **Test expectations:** empty file → missing; `kind:` wrong/absent → missing + diagnostic; valid `kind:`+`provenance:` → present. RED→GREEN.
- **Batch:** 1
