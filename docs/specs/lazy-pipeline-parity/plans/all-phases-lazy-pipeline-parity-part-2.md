  - **`user/scripts/lazy_parity_audit.py`** — importable module + CLI (`--repo-root <path>`, optional `--pair <derived-name>`, exit 0 clean / exit 1 drift). Implements C1–C6 per SPEC §Technical Design §2:
    - **C1** (Tier-1 completeness): every `## Step`/`### sub-step` heading in the canonical has a `headings[]` entry for the pair — else fail naming the heading **and** the pair.
    - **C2** (coverage resolves): a `restated`/`inherited` entry's evidence regex (after token substitution) matches in the derived file.
    - **C3** (Tier-2 predicates): every mechanic in the pair's `mechanic_set` (minus any `mechanic_overrides` whose `coverage=divergence`) matches in the derived file — else fail naming the mechanic id + pair.
    - **C4** (no stale divergence): every `headings[]`/override entry references a heading/mechanic that still exists in the canonical.
    - **C5** (reason hygiene): a `divergence` has a non-empty `reason`; a non-divergence has no `reason`.
    - **C6** (soft/warn → stderr, does NOT affect exit code): a `divergence` with a `doc_anchor` has that anchor text present in the derived skill's prose.
    - **Token substitutions applied before ALL regex matching** (canonical vocab → derived vocab) so axis differences never false-fail.
    - Public surface: an importable `audit_pair(repo_root, pair_name)` returning a list of drift findings (strings), and an `audit_all_pairs(repo_root)` (used by later parts); CLI `__main__` wraps them. **Importable, not `__main__`-only.**

Plus: a `mechanic_override` with `coverage=divergence` correctly **suppresses** C3 for that mechanic on the overriding pair (C3 passes despite the mechanic being absent from the derived file). C6 warns (assert captured in stderr) **without** failing exit code. Each check also has a passing case.
    - *Live zero-drift assertion* for `lazy-batch→lazy-bug-batch`: `assert len(lazy_parity_audit.audit_pair(repo_root, pair_name="lazy-bug-batch")) == 0`. This is the hard gate for the pair; passes iff Phase 1 is complete and WU-1's manifest is correctly populated.
