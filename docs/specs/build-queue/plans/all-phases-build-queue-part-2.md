**Manual Verification Handoff (Jacob) — the substance of this phase:**
```
PHASE 5 FAULT INJECTION — crash-recovery (run by Jacob)
1. KILL WHILE WAITING — kill a queued client; its tickets/<seq>.json is reclaimed (PID-death) by a peer's
   poll; the slot is unaffected; remaining waiters renumber correctly.
2. KILL WHILE HOLDING — kill the wrapper client mid-build; the detached build (its own PID in active.lock)
   survives, completes, writes results/<seq>.json, releases active.lock.
3. STALE LOCK RECLAIM — kill the detached build's PID directly; the next waiter's poll detects the dead
   build_pid, deletes active.lock, proceeds.
4. CORRUPT / PARTIAL STATE — truncate/corrupt active.lock / a ticket / seq.counter; the wrapper tolerates
   it (fail-open / self-heal) rather than wedging the queue.
5. SLOW-BUT-ALIVE NEVER RECLAIMED — a long-but-alive build holds the slot indefinitely (no watchdog, L7).
If any scenario fails, report it; a subagent fixes build-queue.ps1 and you re-run. When all pass, reply
"Phase 5 verified" and the SPEC Status flips to Complete.
```
