### WU-1 — Queue core `build-queue.ps1`

- **Scope (PHASES.md Phase 1 deliverables):**
  - [ ] `user/scripts/build-queue.ps1` implementing the acquire algorithm: atomic seq allocation
    (`CreateNew` on a transient `seq.counter.lock`, bounded retry), ticket enqueue, ~1s poll loop with
    PID-death reclaim of stale tickets/`active.lock`, head-check + `CreateNew`-on-`active.lock` claim,
    detached `Start-Process` run redirected to `logs/<seq>.log`, `build_pid` recorded in `active.lock`,
    log tail to the wrapper's stdout, `results/<seq>.json` write on completion, `active.lock` delete,
    build exit code propagated as the wrapper's exit code.
  - [ ] Wrapper CLI contract: `-Op <msbuild|mstest|nxbuild|nxtest>`, `-Exec <path to filtered script>`,
    trailing pass-through args via `[Parameter(ValueFromRemainingArguments=$true)] $ExecArgs`.
  - [ ] `machine-perf.ps1 -Json` snapshot captured at build start, stored in `active.lock`.
  - [ ] Position heartbeat emitted **only when position changes** (resolves the heartbeat-cadence open
    question by construction).
  - [ ] `~/.claude/state/build-queue/` subtree created on first run (`seq.counter`, `tickets/`,
    `logs/`, `results/`).
- **TDD:** no — there is no PowerShell test framework wired in this repo and the contract (concurrent
  serialization, FIFO, detached-survives-timeout) is only observable across multiple processes/worktrees.
  Verification is the Manual Verification Handoff below, run by Jacob.
- **Files to create/modify:**
  - `C:/Users/JacobMadsen/source/repos/claude-config/user/scripts/build-queue.ps1` — **net-new**
    `[VERIFY: ls "C:/Users/JacobMadsen/source/repos/claude-config/user/scripts/build-queue.ps1"` returns
    not-found today — confirmed absent at planning time]`
- **Implementation goal:** A single PowerShell script matching SPEC §"Acquire algorithm" steps 1–5
  exactly. Reuse, do NOT reinvent:
  - Runs the per-worktree filtered script **unchanged** as the `-Exec` target — one of
    `build-filtered.ps1` / `test-filtered.ps1` / `client-build-filtered.ps1` / `client-test-filtered.ps1`
    `[VERIFY: ls "C:/Users/JacobMadsen/source/repos/Cognito Forms/.claude/scripts/build-filtered.ps1"` —
    all four confirmed present at planning time]`. The wrapper runs it **inside PowerShell**
    (`Start-Process`/`&`), NOT as a Bash tool call — so the Phase-3 enforce hook never sees the inner
    `dotnet`/`nx` and the sanctioned path needs no bypass token.
  - Consumes `machine-perf.ps1 -Json` for the start snapshot
    `[VERIFY: grep -n '\$Json' "C:/Users/JacobMadsen/source/repos/claude-config/user/scripts/machine-perf.ps1"` —
    `$Json` switch at line 14, `if ($Json)` emit at line 147, confirmed]`. Invoke it via its live path
    `$HOME/.claude/scripts/machine-perf.ps1` (symlinked, machine-global).
  - State subtree under `$HOME/.claude/state/build-queue/` (the `LAZY_STATE_DIR` convention — machine-global,
    shared by all four worktrees by construction).
- **Critical correctness invariants (from SPEC §"FIFO + crash-recovery correctness", L3/L7):**
  - Seq numbers strictly increasing & unique → strict arrival order; only the lowest live ticket attempts
    to claim.
  - `CreateNew` on `active.lock` is the real mutex — only one creator wins the residual race → never two
    concurrent holders.
  - `active.lock` records the **detached build's PID**, NOT the wrapper client's PID. This is what makes a
    client (Bash-tool 600s) timeout survivable: the lock is reclaimed only when the *build itself* dies.
  - Stale reclaim is **PID-death only** — NO max-runtime watchdog. A legitimately-slow-but-alive build
    holds the slot until manually killed.
  - The poll loop's reclaim sweep must tolerate a ticket/lock/counter that is truncated or mid-write
    (fail-open / skip-and-continue) rather than throwing — Part 2's Phase 5 will adversarially stress this.
- **Spec requirements:** SPEC §"Acquire algorithm" (verbatim 5-step algorithm), §"Queue state layout"
  (exact file shapes: `seq.counter`, `tickets/<seq>.json` = `{seq,pid,worktree,op,started_wait_at}`,
  `active.lock` = `{seq,build_pid,op,worktree,started_at,log_path,machine_perf}`, `results/<seq>.json` =
  `{seq,exit_code,ended_at}`).
- **Batch:** 1 (solo).
