## Phase: P1 — `build-queue-runner.ps1` (self-releasing build runner)

**Goal:** Add a committed runner script that *is* the detached process: it runs the filtered build/test
script as a nested child, then records its own `results/<seq>.json` and releases `active.lock` (seq-scoped,
idempotent) so the outcome survives the foreground wrapper being killed.
**Entry criteria:** None (net-new file; no `build-queue.ps1` change).
**SPEC.md references:** §"Proposed Fix Direction" (child owns Step 5); §"Open Questions" (both resolved: static committed runner; child releases seq-scoped+idempotent).

### WU-1 — Create self-releasing `build-queue-runner.ps1`

- **Scope:** PHASES.md P1 deliverables (all). New runner script.
- **TDD:** no (no Pester; verified by the scripted repro below).
- **Files to create/modify:** `user/scripts/build-queue-runner.ps1` — **net-new (create).**
- **Batch:** 1.
- **Spec requirements (SPEC §Proposed Fix Direction + PHASES P1):**
  - Params: `-Exec` (abs path, mandatory), `-Seq` (int, mandatory), `-StateRoot` (`[string]`, default `(Join-Path $HOME '.claude\state\build-queue')` — mirror `user/scripts/build-queue-status.ps1:15` `-StateRoot` [VERIFY: `grep -n "StateRoot" user/scripts/build-queue-status.ps1`]), `[Parameter(ValueFromRemainingArguments=$true)] $ExecArgs`.
  - **Run the filtered script as a nested CHILD process** via the call operator:
    `& powershell.exe -NoProfile -ExecutionPolicy Bypass -File $Exec @ExecArgs`, then capture `$LASTEXITCODE`.
    **Do NOT** call `& $Exec` directly — the filtered scripts end with `exit $code` (e.g. `build-queue.ps1` ends `exit $exitCode` at the analogous spot), and a script invoked via `& <path.ps1>` runs `exit` against the **whole runner process**, killing it before bookkeeping. Native `@ExecArgs` splatting re-quotes grandchild args correctly — no manual `Format-ProcArg` needed inside the runner.
  - Build the results dir path as `(Join-Path $StateRoot 'results')` and ensure it exists (`New-Item -ItemType Directory -Force` guarded), matching `build-queue.ps1`'s `$resultsDir` shape [VERIFY: `grep -n '\$resultsDir' user/scripts/build-queue.ps1`].
  - **Idempotent result write:** write `{seq, exit_code, ended_at}` (compress JSON, `(Get-Date).ToString('o')`) to `results/<seq>.json` via atomic `results/<seq>.tmp` → `[System.IO.File]::Replace(..., [NullString]::Value)` with a `WriteAllText` fallback — reuse the exact pattern at `build-queue.ps1:301-308` [VERIFY: `grep -n 'System.IO.File\]::Replace' user/scripts/build-queue.ps1`]. Repeating the write yields identical content (idempotent).
  - **Seq-scoped lock release:** `$activeLock = Join-Path $StateRoot 'active.lock'`. Under `Get-SafeValue` [VERIFY: `grep -n 'function Get-SafeValue' user/scripts/build-queue.ps1`], read+parse `active.lock`; remove it with `Remove-Item -Force` **only if** its parsed `.seq` equals `-Seq`. No-op if absent, unparseable, or seq-mismatched (NEVER delete a successor's lock).
  - `exit` with the captured code (so the wrapper's `$proc.ExitCode` stays the real build code).
  - House style per MANDATORY RULE 9.
- **Implementation goal:** A standalone, side-effect-correct runner provable in isolation: given a stub `-Exec` and a seeded `-StateRoot`, it writes the right result, releases only its own lock, and propagates the exit code — with the stub's own `exit` NOT aborting the bookkeeping.
- **Repro gate (orchestrator runs after the subagent; paste real output):**
  1. **Setup** a temp state root, e.g. `$T = Join-Path $env:TEMP ("bqr-" + [guid]::NewGuid())`; create `$T/results`; write `$T/active.lock` = `{"seq":42,...}`; write a stub exec `$T/stub.ps1` that does `Start-Sleep 1; exit 7`.
  2. Run: `powershell -NoProfile -ExecutionPolicy Bypass -File user/scripts/build-queue-runner.ps1 -Exec $T/stub.ps1 -Seq 42 -StateRoot $T`.
  3. **Assert:** runner exit code = `7`; `$T/results/42.json` exists with `exit_code:7`; `$T/active.lock` is **gone**.
  4. **Seq-guard:** repeat with `active.lock` seq = `99` and `-Seq 42` → after run, `$T/active.lock` is **untouched**; `42.json` still written.
  5. **Idempotency:** re-invoke the result-write path for seq 42 (re-run step 2) → `42.json` well-formed, content stable.
  6. **Nested-exit:** confirm steps 2–3 prove the stub's `exit 7` did NOT prevent the result/lock bookkeeping (it ran to completion).

**Integration Notes for WU-2 (carry forward):**
- Nested-child invocation is load-bearing — WU-2's wrapper must NOT re-collapse it into a direct call.
- The runner stays alive for the whole build → its PID is a valid `build_pid` for stale-reclaim; WU-2 records `build_pid = $proc.Id` where `$proc` is the runner.
- The runner owns the **canonical** result+release; WU-2's demoted Step 5 reuses the SAME idempotent-write + seq-scoped-removal so wrapper+runner can never double-act.

## Phase: P2 — Wire `build-queue.ps1` to the runner + demote Step 5

**Goal:** Point the wrapper's detached launch at `build-queue-runner.ps1`, keep the live-output tail loop,
and demote the wrapper's Step 5 to an idempotent, seq-scoped best-effort fallback. After this phase the
orphan path is fixed end-to-end.
**Entry criteria:** P1 complete (`build-queue-runner.ps1` exists and its repro gate passed).
**SPEC.md references:** §"Affected Area" (wrapper launch + Step 5); §"Proposed Fix Direction" (wrapper no longer load-bearing).

### WU-2 — Wire `build-queue.ps1` to the runner + demote Step 5

- **Scope:** PHASES.md P2 deliverables (all).
- **TDD:** no (verified by the kill-mid-build + regression repro below).
- **Files to create/modify:** `user/scripts/build-queue.ps1` — modify Step 4 launch (`283-308`) and Step 5 (`360-376`); tail loop (`317-358`) unchanged.
- **Batch:** 1 (separate phase from WU-1; no file overlap with the runner).
- **Spec requirements:**
  - **Step 4 launch (`283-288`)** [VERIFY: `grep -n "Start-Process -FilePath 'powershell.exe'" user/scripts/build-queue.ps1`]: build the `-ArgumentList` string with the existing `Format-ProcArg` [VERIFY: `grep -n 'function Format-ProcArg' user/scripts/build-queue.ps1`] so it invokes `build-queue-runner.ps1` (resolve its path as a sibling of this script, e.g. `Join-Path $PSScriptRoot 'build-queue-runner.ps1'`) with: `-Exec (Format-ProcArg $Exec)`, `-Seq $seq`, `-StateRoot (Format-ProcArg $stateRoot)`, then the forwarded `$execArgsArr` each via `Format-ProcArg`. `$proc` is now the runner; keep `$buildPid = $proc.Id` and the `active.lock` write (`292-308`) unchanged — `build_pid` is the runner PID, so stale-reclaim semantics are preserved.
  - **Tail loop (`317-358`) unchanged** — the wrapper still streams the runner's stdout/err (carrying the filtered script's output) to its own stdout.
  - **Step 5 (`360-376`)** [VERIFY: `grep -n 'Remove-Item \$activeLock' user/scripts/build-queue.ps1`] demoted to **idempotent + seq-scoped best-effort**: keep `$exitCode = $proc.ExitCode`; write `results/<seq>.json` via the same atomic tmp→Replace (matching the runner); make the `active.lock` removal seq-scoped (read+parse, remove only if `.seq == $seq`), all under `Get-SafeValue`. Keep `exit $exitCode`. The runner will usually have already written/released; the wrapper's repeat is harmless.
- **Implementation goal:** The detached runner is now the canonical recorder/releaser; the surviving wrapper is a redundant, conflict-free best-effort. A killed wrapper no longer strands the result or the lock.
- **Repro gate (orchestrator runs after the subagent; paste real output):**
  1. **Orphan path (the fix):** create a stub exec that sleeps ~25s then `exit 1`. Launch via the wrapper (`build-queue.ps1 -Op msbuild -Exec <stub>`) as a background process; once `active.lock` appears, **kill the wrapper process** (not the runner). Let the runner finish. **Assert:** `results/<seq>.json` written with `exit_code:1`; `active.lock` released — with no wrapper alive to have done it.
  2. **Happy path (regression):** run the wrapper against stubs exiting `0` and `1` to completion. **Assert:** exactly ONE `results/<seq>.json` with the correct code; `active.lock` released once; wrapper exit code matches.
  3. **Concurrency (regression):** launch the wrapper from two different cwins concurrently against sleep stubs. **Assert:** serialized (no overlapping START/END), FIFO seq order, both results recorded, both slots released.
  4. **Stale-reclaim still works:** kill the runner (`build_pid`) outright → a waiter reclaims ~3 ticks after the PID dies (unchanged).
  5. **Status reporter sanity:** run `build-queue-status.ps1` after the orphan repro → clean completed/idle render (no orphaned seq).

**Integration Notes:** Final WU. The four routing skills already poll `results/<seq>.json` for the background outcome; that file is now reliably written by the runner regardless of wrapper survival — **no skill change required**. The 10-min Bash-timeout mitigation (commit `5bdd74e`) remains a valid frequency-reducer, orthogonal to this correctness fix.
