### Phase: copy-lock P2 — Copy-lock prevention (reap the prior-run locker before the copy)

**Goal:** Kill the leftover `testhost`/`dotnet` process holding a handle on the worktree's `bin/Debug` DLLs *before* the build's copy step, so the copy succeeds instead of failing MSB3027. Scoped to the worktree's own artifacts — never a global kill (honors the hygiene work's Locked Decision 1).
**Entry criteria:** Part 1 (Phase 1) committed — WU-4/WU-5 edit the same `build-queue-hygiene.ps1` / `build-queue-runner.ps1` files (one-writer-per-file; parts execute strictly in order).
