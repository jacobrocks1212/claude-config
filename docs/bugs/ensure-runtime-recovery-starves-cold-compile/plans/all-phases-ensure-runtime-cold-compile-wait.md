## Blocking Issue Protocol

If a blocking issue is encountered at any point during execution:

1. **Stop all in-progress work.** Do not launch new subagents.
2. **Commit and push any completed phases** not yet committed.
3. **Print a blocking-issue report** (Completed phases + hashes; Blocked phase; Reason; Recovery suggestion; Remaining phases).
4. **Do not work around the blocker.** Write `BLOCKED.md` per the sentinel schema and halt.

Blocking issues include: a smoke-suite/parity-audit failure that can't be fixed after 2 retries; a git push conflict unresolvable by rebase; an entry criterion referencing work outside this plan; any error requiring architectural decisions beyond the SPEC's Locked Decisions.
