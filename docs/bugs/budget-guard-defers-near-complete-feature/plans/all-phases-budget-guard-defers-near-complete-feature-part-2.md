#### WU-6 — End-of-run near-complete resume flush
- **Scope:** PHASES.md Phase 3 deliverables (re-scan deferred for near-complete, resume in queue order, terminal only when none resumable, never resume evicted).
- **TDD:** yes (via `lazy-state.py --test` fixtures characterized in `test_lazy_core.py`).
- **Files to modify:** `user/scripts/lazy-state.py` — the `current is None` budget terminal block, before the `queue-exhausted-budget-deferred` return [VERIFY: grep -n "queue-exhausted-budget-deferred" user/scripts/lazy-state.py].
- **Test files:** `user/scripts/test_lazy_core.py` and/or `lazy-state.py --test` fixtures; regenerate `tests/baselines/lazy-state-test-baseline.txt` through `_normalize_smoke_output` [VERIFY: grep -n "_normalize_smoke_output" user/scripts/test_lazy_core.py] (never by hand).
- **Test expectations:** A feature in `budget_deferred_skipped` [VERIFY: grep -n "budget_deferred_skipped" user/scripts/lazy-state.py] that is now near-complete is DISPATCHED at the flush (probe sets `budget_resumed_near_complete`), not returned as the terminal. When NO deferred feature is near-complete, `queue-exhausted-budget-deferred` STILL fires (terminal unchanged for the genuine case). An evicted feature (`_bg_evicted` [VERIFY: grep -n "_bg_evicted" user/scripts/lazy-state.py]) is NEVER resumed. Multiple near-complete deferred features resume one-per-probe in queue order.
- **Implementation goal:** Before the terminal return at the `current is None` block, iterate `budget_deferred_skipped` in queue order; for the first whose `lazy_core.feature_is_near_complete(spec_path, repo_root)` is True AND which is not in `_bg_evicted`, set it as the dispatch target (resume to validation) and add a `_diag` audit line + a `budget_resumed_near_complete: <feature_id>` probe field. The terminal fires only when none qualifies.
- **Spec requirements:** SPEC `## Fix Direction` item 2 (end-of-run resume flush) + Theory 3.
