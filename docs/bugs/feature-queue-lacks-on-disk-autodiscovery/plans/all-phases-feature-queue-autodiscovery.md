- **Files to create/modify:**
  - `user/scripts/lazy-state.py` — add `feature_tier(spec_path: Path) -> int`; add `_find_open_feature_dirs(features_dir: Path, queued_ids: set[str]) -> list[Path]`; extend `load_queue` (~307) to merge discovered entries when `data.get("autodiscover") is True`; reconcile the `queue-missing` early-return (~1350) so a non-empty merged list (empty explicit queue + disk discovery) does NOT short-circuit to `queue-missing`; add the seven `_build_fixture` cases + their assertions in `run_smoke_tests`.

- **Test expectations (author these fixtures FIRST):**
  - `autodiscover-off` — feature dir `feat-ad` on disk (`SPEC.md` `**Status:** Draft`), NOT in `queue.json`, NO `autodiscover` flag, queue has one OTHER explicit feature ⇒ `feat-ad` is INVISIBLE (dispatch is for the explicit feature only; byte-identical to today).
  - `autodiscover-on` — same `feat-ad` dir WITH top-level `"autodiscover": true` ⇒ `feat-ad` is discovered and dispatched (Draft SPEC routes to `/spec`).
  - `autodiscover-excludes-complete` — a `Complete` dir with a valid `COMPLETED.md` receipt + `autodiscover: true` ⇒ NOT re-enqueued (queue exhausts past it).
  - `autodiscover-surfaces-receiptless-complete` — a `Complete` dir with NO `COMPLETED.md` + `autodiscover: true` ⇒ surfaced (→ `terminal_reason: "completion-unverified"`), with a `_diag` mentioning the dir. [VERIFY: grep -n "completion-unverified" user/scripts/lazy-state.py]
  - `autodiscover-dedupes-explicit-twin` — `feat-dup` BOTH explicitly queued AND on disk + `autodiscover: true` ⇒ appears once; explicit entry wins and is listed first.
  - `autodiscover-orders-by-priority` — two discovered dirs, one SPEC `**Priority:** P1`, one `**Priority:** P3` (or absent) ⇒ P1 sorts before P3/absent; equal priority falls back to directory name.
  - `autodiscover-empty-queue-not-missing` — `"queue": []` + `"autodiscover": true` + one open disk dir ⇒ NOT `queue-missing`; the disk dir is dispatched. (This is claude-config's own live scenario — `queue.json` is `{"queue": []}`.)
- **Implementation goal:**

  - `_find_open_feature_dirs` mirrors `bug-state.py::_find_open_bug_dirs` [VERIFY: grep -n "def _find_open_bug_dirs" user/scripts/bug-state.py]: iterate `features_dir` one level deep; skip non-dirs, `_`-prefixed dirs, and any dir whose name is in `queued_ids`; require `SPEC.md`; read `**Status:**` via `lazy_core.spec_status`; EXCLUDE `Superseded`; EXCLUDE `Complete` WHEN `has_completion_receipt(child)` is true [VERIFY: grep -n "def has_completion_receipt" user/scripts/lazy_core.py]; a `Complete`-WITHOUT-receipt dir is NOT skipped (append + emit a `_diag` — the receipt gate downstream fires `completion-unverified`, exactly as the bug loader surfaces a receiptless `Fixed`); sort by `(feature_tier(spec), child.name)`.
  - `load_queue` [VERIFY: grep -n "def load_queue" user/scripts/lazy-state.py]: after parsing `items = data.get("queue", [])`, if `data.get("autodiscover") is True`, build `queued_ids = {e.get("id") for e in items if isinstance(e, dict)}`, call `_find_open_feature_dirs(repo_root / "docs" / "features", queued_ids)`, and APPEND one discovered entry per dir of shape `{"id": dirname, "name": <SPEC '# ' title or dirname>, "spec_dir": dirname, "tier": feature_tier(spec_dir_path), "queue_entry": None}`. Return `items + discovered`. When the flag is absent/falsy, return `items` UNCHANGED (byte-identical). The discovered entry MUST carry the raw-queue-item keys `id`/`name`/`spec_dir`/`tier` because the `compute_state` walk loop reads `entry.get("spec_dir")` directly [VERIFY: grep -n 'entry.get("spec_dir")' user/scripts/lazy-state.py] — do NOT return the bug loader's normalized `spec_path` shape here (the two loaders' return shapes legitimately differ).
