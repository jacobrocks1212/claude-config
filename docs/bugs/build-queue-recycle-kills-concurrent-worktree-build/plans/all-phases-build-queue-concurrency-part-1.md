### Phase: build-queue-recycle‚Ä¶ P1 ‚Äî Close Vector A (atomic provisional lock write + reclaim hardening)

**Goal:** Eliminate the stale-lock-reclaim race that can admit a second concurrent queued build ‚Äî write the provisional lock atomically, and advance the stale counter only on a *confirmed-dead* pid (treat an unreadable `'unknown'` read as inconclusive).
**Entry criteria:** None (first phase).
**SPEC.md references:** ¬ßReproduction Steps "Vector A"; ¬ßEvidence "Serialization hole (Vector A)"; ¬ßOpen Questions (atomic-write invariant as the assertable guard). PHASES.md Phase 1.

#### WU-1 ‚Äî Atomic provisional lock write + confirmed-dead-only reclaim

- **Scope (PHASES.md Phase 1 deliverables):**
  - [ ] Provisional lock claim (`build-queue.ps1:202-224`) writes atomically ‚Äî build the body to a temp file (`active.$seq.tmp`) and `[System.IO.File]::Replace` / move into place, mirroring the final-write pattern already at `build-queue.ps1:301-317`, instead of the raw `Write`/`Flush` into the `CreateNew` handle. The `FileMode::CreateNew` mutual-exclusion guarantee (only one claimant creates the lock) MUST be preserved.
  - [ ] `Get-ActiveLockStatus` (`build-queue.ps1:162-176`) / the reclaim loop (`build-queue.ps1:182-200`) no longer count a single `'unknown'` read toward `staleThreshold`. Reclaim advances only on `'dead'` (pid confirmed not alive via `Test-PidAlive`); `'unknown'` is inconclusive (hold or reset the tick, do not increment); `'absent'` still means free-to-claim.
  - [ ] Reads that can race the claim window (`build-queue.ps1:165`, `build-queue.ps1:391`, and the runner read at `build-queue-runner.ps1:196`) tolerate a transient partial/locked read without misclassifying ‚Äî a bounded re-read so a mid-write observation resolves to the real status, not `'unknown'`.
- **TDD:** yes.
- **Files to create/modify:**
  - `user/scripts/build-queue-hygiene.ps1` ‚Äî add three **pure decision helpers** (see directive): `Get-ActiveLockStatusFromText`, `Test-ShouldReclaimLock`, `Set-LockFileAtomic`.
  - `user/scripts/build-queue.ps1` ‚Äî wire the helpers into the poll loop; atomic provisional write; bounded re-read in `Get-ActiveLockStatus` (`:162-176`); reclaim advances only on `'dead'` (`:191-197`); tolerant read at the release-time lock read (`:389-396`).
  - `user/scripts/build-queue-runner.ps1` ‚Äî tolerant re-read at the release-time lock read (`:194-202`).
- **Test files:** `user/scripts/build-queue-hygiene.Tests.ps1` (extend ‚Äî new `Describe` blocks for the pure helpers).
- **Test expectations (write these FIRST):**
  1. **Lock body round-trip:** `Set-LockFileAtomic` writes a JSON lock body to a temp path and a subsequent read yields the **intact** JSON (round-trips; no truncation). (SPEC Open Question: the atomic-write invariant is the assertable guard.)
  2. **Classification correctness:** `Get-ActiveLockStatusFromText` returns `'alive'` for valid JSON with a live pid (inject a liveness probe returning `$true`), `'dead'` for valid JSON with a not-alive pid (probe `$false`), and `'unknown'` for empty/garbage/partial text (unparseable). A merely-truncated body classifies `'unknown'` at the text level ‚Äî and the reclaim helper (below) must NOT treat that as reclaimable.
  3. **Confirmed-dead-only reclaim:** `Test-ShouldReclaimLock` returns `$true` only after `staleThreshold` consecutive `'dead'` observations while this seq is lowest; an `'unknown'` in the sequence does NOT count toward the threshold (holds/resets), and `'alive'`/`'absent'` reset it. Assert an all-`'unknown'` sequence never reclaims, and a `dead,unknown,dead,dead` sequence (threshold 3) does not reclaim on the interrupted run.
- **Implementation goal:** Make the lock-protocol invariants unit-testable via the repo's existing Pester harness WITHOUT executing `build-queue.ps1`'s top-level queue side effects. `build-queue.ps1` is a top-to-bottom script (not dot-sourceable without running the queue), so the *pure* decision logic is extracted into `build-queue-hygiene.ps1` ‚Äî the already-dot-sourced, side-effect-free function module that already hosts a pure-decision helper (`Test-BuildLogFailure` [VERIFY: `grep -n "function Test-BuildLogFailure" user/scripts/build-queue-hygiene.ps1`]). `build-queue.ps1` calls these helpers from its poll loop. **Robustness constraint:** the wrapper must keep working if the hygiene dot-source fails ‚Äî `Get-ActiveLockStatus` calls the helper when available and otherwise falls back to inline classification (never let a hygiene load failure break the queue's core lock path; the hygiene dot-source at `build-queue.ps1:47-49` is already `Get-SafeValue`-wrapped/fail-open [VERIFY: `grep -n "build-queue-hygiene.ps1" user/scripts/build-queue.ps1`]). Reuse, do NOT reinvent: the atomic-write helper `Set-LockFileAtomic` generalizes the existing final-write temp+`File.Replace` idiom at `build-queue.ps1:310-317`; `Get-ActiveLockStatusFromText` receives an injected liveness probe so tests need no real pids, mirroring how the current `Get-ActiveLockStatus` uses `Test-PidAlive` [VERIFY: `grep -n "function Test-PidAlive" user/scripts/build-queue.ps1`]. Preserve the `CreateNew` exclusive claim ‚Äî the atomic write replaces only the *body-write* step after the claim, not the claim itself.
- **Spec requirements:** SPEC ¬ßEvidence "The provisional lock is written with a raw `Write`/`Flush` ‚Ä¶ not the atomic `File.Replace` used for the final lock ‚Ä¶ readers open `FileShare.ReadWrite`, so a reader in the sub-second window can read a truncated lock ‚Üí `'unknown'`"; "Stale-lock reclaim ‚Ä¶ deletes the lock after 3 `'unknown'`/`'dead'` ticks". ### Phase: build-queue-recycle‚Ä¶ P2 ‚Äî Occupancy-gate the VBCSCompiler recycle

**Goal:** Make `Reset-CompilerServer` fire only when this build is the sole known-active queue seq, so a build finishing in worktree A cannot force-kill the shared VBCSCompiler a concurrent queued build in worktree B is using. Shared compilation stays enabled (Locked Decision).
**Entry criteria:** Phase 1 complete (occupancy read is against a trustworthy `active.lock`; queued-concurrent case already largely prevented).
**SPEC.md references:** ¬ßReproduction "Vector B"; ¬ßProven Findings; PHASES.md Phase 2 + Locked Decision "occupancy-gate the recycle".

**Plan-specific execution note (Phase 2 design decision ‚Äî occupancy helper home + signature):** The occupancy query lives in **`build-queue-hygiene.ps1`** as a new helper `Get-BuildQueueOccupancy` ‚Äî both the wrapper (`build-queue.ps1`) and the detached runner (`build-queue-runner.ps1`) already dot-source that module [VERIFY: `grep -n "build-queue-hygiene.ps1" user/scripts/build-queue-runner.ps1`], and the runner has no queue-state accessor of its own. The helper is self-contained (does its own tickets + `active.lock` read under `-StateRoot`, its own inline pid-liveness fail-safe-to-alive check) so it carries no cross-file dependency and stays fail-open. `Reset-CompilerServer` gains an **occupancy signal parameter** (a `[bool]$OtherBuildActive` / occupancy-count param) ‚Äî the caller computes occupancy and passes it; the function stays fail-open and, when another build is active, **skips** both the `dotnet build-server shutdown` (`:380`) and the `Stop-Process -Force VBCSCompiler` (`:388-392`) and reports why.
