### Phase: build-queue-recycle… P3 — Bypass ergonomics (tolerate a command-prefixed BUILD_QUEUE_BYPASS)

**Goal:** Fix the asymmetry where the deny surface is unanchored (a build is caught anywhere in the command) but the bypass token is leading-anchored, so a legitimate `cd "…" && BUILD_QUEUE_BYPASS=1 …` is denied. Recognize the bypass when it leads the *build segment* (after a `cd …&&`, `;`, or pipeline prefix), and fix the misleading deny message — WITHOUT re-opening the enforcement escape the `cd-prefix-bypass` fix closed.
**Entry criteria:** None (independent of Phases 1–2 and Phase 4).
**SPEC.md references:** §Secondary defect (coupled — bypass ergonomics); §Verified Symptoms #3; PHASES.md Phase 3.

Keep the Cognito-worktree scope gate and the fail-open contract intact.
  - [ ] Deny messages (`_redirect_reason`, `:175-239`) state the constraint explicitly — that the recognized bypass forms include a `cd …&&`-prefixed token (not only a *leading* token) — so an agent's first bypass attempt succeeds.

  - [ ] Tests: `cd … && BUILD_QUEUE_BYPASS=1 <build>` → allowed; `cd … && <build>` (no token) → denied; bare `BUILD_QUEUE_BYPASS=1 <build>` → allowed (unchanged); `NAME=val BUILD_QUEUE_BYPASS=1 <build>` → allowed (unchanged).

- **Test expectations (write FIRST):** The runner feeds a JSON payload (`{"tool_name":"Bash","cwd":"<forced-cognito>","tool_input":{"command":"<case>"}}`) to the hook on stdin and asserts on stdout. Cases (with `<build>` = `powershell.exe -File .../build-filtered.ps1`):
  1. `cd "C:/…/Cognito Forms" && BUILD_QUEUE_BYPASS=1 <build>` → **allow** (stdout empty / no `"permissionDecision": "deny"`), exit 0.
  2. `cd "C:/…/Cognito Forms" && <build>` (no token) → **deny** (stdout contains `"permissionDecision": "deny"` and a `/msbuild`-or-`/nxbuild` redirect).
  3. `BUILD_QUEUE_BYPASS=1 <build>` (bare leading token) → **allow** (unchanged behavior).
  4. `NAME=val BUILD_QUEUE_BYPASS=1 <build>` (env-prefixed leading token) → **allow** (unchanged behavior).

  6. **Fail-open:** malformed JSON on stdin → **allow** (exit 0, no deny) — proves the fail-open contract survives.

- **Implementation goal:** Widen bypass recognition from "leading env-assignment prefix only" to "leads the build segment." Today `_BYPASS_RE` [VERIFY: `grep -n "_BYPASS_RE" user/hooks/build-queue-enforce.sh`] is `^\s*(?:[A-Za-z_][A-Za-z0-9_]*=\S+\s+)*BUILD_QUEUE_BYPASS=1(?:\s|$)` and is applied once against the whole command via `_BYPASS_RE.match(command)` [VERIFY: `grep -n "_BYPASS_RE.match" user/hooks/build-queue-enforce.sh`]. Split the command into segments on `&&`, `;`, and `|` (mirror how the deny side reasons about the whole command) and treat the bypass as present if the token leads **any** segment (after an optional `cd …` and/or `NAME=value` prefix within that segment). Equivalent single-regex option: allow an optional leading `cd …(&&|;)` (and pipeline) prefix before the existing env-assignment/token pattern. **Do NOT** loosen it to "token anywhere" — that would let `dotnet build && echo BUILD_QUEUE_BYPASS=1` slip through; the token must LEAD a segment. Preserve: the `_WRAPPER_RE` early-allow, the `_is_cognito_worktree` scope gate, the safe-variant suppression, and the fail-open `try/except → _allow()` wrapper. Fix the deny-message "Emergency one-off" lines [VERIFY: `grep -n "Emergency one-off" user/hooks/build-queue-enforce.sh`] to show the `cd "…" && BUILD_QUEUE_BYPASS=1 …` form. For the **test runner's scope determinism**, prefer one of: (a) point `cwd` at a throwaway git repo the runner `git init`s with `remote.origin.url=https://github.com/cognitoforms/cognito.git` in a temp dir, or (b) if simpler and robust, have the runner assert on cases where the deny/allow decision is independent of scope by exercising the hook's Python body directly with a stubbed `_is_cognito_worktree`. Option (a) is closest to production and recommended; whichever is chosen, the runner must be hermetic (no dependence on the executor's actual repo/remote) and clean up its temp dir.
- **Spec requirements:** SPEC §Secondary defect ("`_BYPASS_RE` … matches only an optional `NAME=value` prefix before `BUILD_QUEUE_BYPASS=1`, not a `cd "…" &&` prefix … a legitimate … `cd … && BUILD_QUEUE_BYPASS=1 …` is now denied"); PHASES.md Phase 3 "Minimum Verifiable Behavior".
