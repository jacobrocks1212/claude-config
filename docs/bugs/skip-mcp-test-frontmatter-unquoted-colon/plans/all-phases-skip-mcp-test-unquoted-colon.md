### Phase: skip-mcp-test-frontmatter-unquoted-colon P1 — Tolerant read at `parse_sentinel`

**Goal:** An unquoted colon-space (or trailing-colon) in a flat scalar frontmatter *value* is read as a literal instead of hard-halting the state script; strict schema + `_die`-on-genuine-malformation are preserved.
**Entry criteria:** none (first phase).
**SPEC.md references:** `## Problem`; `## Verified Symptoms` (1); `## Recommended Fix Direction` → "Tolerant read (primary)"; `## Affected Area` row 1.

#### WU-1 — Tolerant read at `parse_sentinel` + unit regression tests

- **Scope (PHASES P1 deliverables):**
  - [ ] New module-level tolerant-load helper in `lazy_core.py` (e.g. `_yaml_load_tolerant(yaml_body) -> dict | None`).
  - [ ] `parse_sentinel`'s `except yaml.YAMLError` arm calls the helper before `_die`.
  - [ ] Unit tests (tolerance + control + non-vacuity + byte-identical well-formed).
- **TDD:** yes.
- **Files to create/modify:**
  - `user/scripts/lazy_core.py` — add `_yaml_load_tolerant`; wire it into `parse_sentinel` [VERIFY: `grep -n "def parse_sentinel" user/scripts/lazy_core.py`] at its `except yaml.YAMLError as exc:` arm [VERIFY: `grep -n "except yaml.YAMLError" user/scripts/lazy_core.py`], which currently calls `_die` [VERIFY: `grep -n "def _die" user/scripts/lazy_core.py`].
- **Test files:**
  - `user/scripts/test_lazy_core.py` — co-locate near the existing `parse_sentinel` tests [VERIFY: `grep -n "def test_parse_sentinel" user/scripts/test_lazy_core.py`].
- **Test expectations (write FIRST, must be RED against current `_die` behavior):**
  1. `reason: untestable on this host: no real audio device` (unquoted colon-space) → `parse_sentinel` returns a dict whose `reason` is the literal full string, NO exception/SystemExit.
  2. `skipped_by:` value naming a `key: value` pair → parses to the literal string (no nested mapping, no `_die`).
  3. A trailing-colon value → parses to a literal.
  4. **Control:** `reason: build:step` (colon with NO following space) → still a plain scalar, byte-identical result to today.
  5. **Non-vacuity:** a genuinely-malformed frontmatter that is NOT a flat-scalar colon case (e.g. a broken indented block / a value that yields a non-mapping) → STILL raises `_die`/SystemExit. The tolerant path must NOT mask real malformation.
  6. A well-formed, no-colon sentinel → result byte-identical to pre-change (regression guard).
- **Implementation goal:** In `parse_sentinel`, keep the initial `yaml.safe_load` (well-formed files stay byte-identical — the tolerant path is reached ONLY on `YAMLError`). In the `except yaml.YAMLError` arm, call `_yaml_load_tolerant(yaml_body)`: it operates per-line over the frontmatter, and for each flat top-level `^\s*[A-Za-z0-9_-]+:\s.*$` line whose value is not already quoted, not a flow collection (`[`/`{`), and not a block-scalar indicator (`|`/`>`), single-quotes the value (escaping embedded single quotes) and re-invokes `yaml.safe_load`. If the tolerant re-parse succeeds and yields a `dict`, return it; otherwise fall through to the original `_die` (unchanged final behavior). Do NOT alter `_die`, `skip_waiver_refusal`, or any caller.
- **Spec requirements:** SPEC `## Recommended Fix Direction` — "make `parse_sentinel` pre-process frontmatter so an unquoted colon-space in a *scalar value* is treated as a literal (quote-on-read), keeping strict schema semantics for keys/kinds." SPEC `## Verified Symptoms` (1) and Theory 1 (Confirmed).
- **Batch:** 1.
