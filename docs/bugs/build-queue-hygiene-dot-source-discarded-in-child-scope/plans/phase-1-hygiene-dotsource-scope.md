### Phase: build-queue-hygiene-dot-source-discarded-in-child-scope P1 — Move the hygiene dot-source out of the `Get-SafeValue` child scope in all three callers

**Goal:** Fix the single root cause — the hygiene dot-source runs inside `Get-SafeValue`'s `& $Block` child scope in all three callers, so every hygiene function is discarded and undefined in each caller's real script scope. Move each dot-source to a top-level `try { … } catch { }` (script-scope preserving, still fail-open), correct the stale status-view comment, and add a structural regression guard that goes RED if anyone re-wraps the dot-source.

**Entry criteria:** None (Prerequisites: None per PHASES). Clean working tree.

**SPEC.md references:** "Verified Symptoms" §5 (traced root cause); "Reproduction Steps" → Minimal cause isolation; "Proven Findings" 1–6; "Affected Area" table (three PRIMARY rows + the Regression guard row); "Open Questions" (keep fail-open + add the guard).

**Test files:** same file (this is a test-authoring WU).

**Test expectations — what the new `Describe` block asserts:**
- For EACH caller in `@('build-queue.ps1','build-queue-runner.ps1','build-queue-status.ps1')` (resolved via `Join-Path $PSScriptRoot <name>`), the block parses the caller's SOURCE with `[System.Management.Automation.Language.Parser]::ParseFile($callerPath, [ref]$null, [ref]$null)` and asserts:
  1. **There is exactly one dot-source of `build-queue-hygiene.ps1`** in the file (guards against a stray second copy — SPEC blast-radius audit found exactly one per file).
  2. **That dot-source statement is a TOP-LEVEL statement** — its nearest enclosing statement list is the file's top-level `EndBlock`/`BeginBlock`, NOT a nested `ScriptBlockExpressionAst` that is an argument to a `Get-SafeValue` (or a bare `& { … }`) command invocation.
- **Mechanism (preferred — AST):** use `$ast.FindAll({ param($n) $n -is [System.Management.Automation.Language.CommandAst] -and $n.InvocationOperator -eq [System.Management.Automation.Language.TokenKind]::Dot }, $true)` to locate dot-source `CommandAst` nodes, filter to those whose command text references `build-queue-hygiene.ps1`, then walk the node's `.Parent` chain. Assert NO ancestor on that chain is a `ScriptBlockExpressionAst` (which is how a scriptblock arg to `Get-SafeValue` / `& {}` shows up). Equivalently: the first `StatementBlockAst`/`NamedBlockAst` ancestor must be one of the script's own top-level named blocks (`EndBlock`/`BeginBlock`/`ProcessBlock`), reached WITHOUT passing through a `ScriptBlockExpressionAst`.
  - **Text-based fallback (acceptable if AST proves awkward in the harness):** read each caller's raw lines, find the line matching `\.\s*\(Join-Path \$PSScriptRoot 'build-queue-hygiene\.ps1'\)`, and assert it is NOT the sole statement of a `{ … }` block passed as an argument (e.g. the dot-source line sits at the script's baseline top-level indent and is not immediately preceded within its enclosing brace by an unclosed `Get-SafeValue {` / `& {` opener). Document in a comment that this is the fragile fallback; prefer AST.
- The block must **NEVER dot-source or invoke** the three caller scripts — it reads their source text only.
- Give each assertion a per-caller `It` (e.g. `It 'dot-sources build-queue-hygiene.ps1 at script scope, not inside Get-SafeValue (<caller>)'`) so a single failing caller is named.
