### Phase: P3 — Build-output fidelity gate (Root Cause B)

**Goal:** Add a positive build-output classifier and force exit code / fidelity when an exit-0 build produced no output, so a silently-broken build reports `RESULT=FAIL` / `build_fidelity: no-output` instead of a clean PASS. Surface the new fidelity in the banner, the status view, and the build skills.

**Entry criteria:** Part 1 Complete (per-project sweep + `Read-WithRetry` both merged and Pester-green).

**SPEC.md references:** Proven Finding 2; Root cause B (`Evidence → Source Code`); Theory 2 (Confirmed); Open Questions "Build-output detection method" (resolved: empty/whitespace/near-empty-log default, DLL-touched as a documented follow-on knob) and "New fidelity value naming" (resolved: reuse `no-output`).

**Verified reality (grounding):** for a build op, `build_fidelity` is set only in the `if ($isBuildOp)` block at `build-queue-runner.ps1:137-153` — the `else` at 152 labels everything not caught by `Test-BuildLogFailure` as `verified`; the log guard at 139 tests path/existence only (never size). The log-failure-override mechanism to mirror is 146-149 (`$exitCode = 1; $buildFailed = $true; $buildFidelity = 'log-failure-override'`). The per-project sweep fires only when `$buildFailed` is already true (`:161-163`, guarded `$buildFailed -and $Worktree`). Schema doc-comment for `build_fidelity` is `build-queue-runner.ps1:30` (`"log-failure-override" | "verified" | "n/a"`). `Format-BuildQueueBanner` is `build-queue-hygiene.ps1:1190`; its FAIL arm at 1249 already fires on `$ExitCode -ne 0` (so a forced exit=1 yields FAIL automatically); the next-action suffix arm is 1272-1279 (`NO-TESTS-MATCHED` / exit-4 stale / else `read logs/$Seq.build.err.log`). `Format-BuildQueueBanner` already takes a `$BuildFidelity` param. `build-queue-status.ps1` highlights are at 182-187 (`$buildFidelityStr -eq 'log-failure-override'` Red; `$fidelityStr -eq 'no-output'` Yellow; else plain) — `$buildFidelityStr` is already computed.

#### WU-1 — Build-output classifier + banner no-output arm

- **Scope (PHASES deliverables 1, 3 + their tests):**
  - [ ] Build-output classifier function added to `user/scripts/build-queue-hygiene.ps1` (dot-sourced/testable): given the build-log path (read via Part 1's `Read-WithRetry`), decide whether the build produced output. **Default detection:** an exit-0 build op with a missing / empty / whitespace-only / near-empty build log is classified **no-output**. The stronger expected-output-DLL check (DLL exists AND newer than sources) is documented as a follow-on knob in the function's doc-comment and is **NOT** implemented here [VERIFY: `grep -n "function Read-WithRetry" user/scripts/build-queue-hygiene.ps1` — must exist from Part 1].
  - [ ] `Format-BuildQueueBanner` (`build-queue-hygiene.ps1:1190`; FAIL arm 1249; next-action suffix 1272-1279): add a dedicated next-action arm for `$BuildFidelity -eq 'no-output'` (e.g. `'build produced no output; delete obj/bin and rebuild'`), parallel to the existing guidance. `RESULT=FAIL` already falls out of the forced exit=1 via the existing `$ExitCode -ne 0` arm — do NOT add a new RESULT label [VERIFY: `grep -n "function Format-BuildQueueBanner" user/scripts/build-queue-hygiene.ps1`].
- **TDD:** yes.
- **Files to create/modify:** `user/scripts/build-queue-hygiene.ps1` (classifier fn + banner arm).
- **Test files:** `user/scripts/build-queue-hygiene.Tests.ps1` (new `Describe` for the classifier + extend the `Format-BuildQueueBanner` `Describe`).
- **Test expectations:**
  - Classifier returns **no-output** for: a missing log path, an empty (0-byte) log, a whitespace-only log, and a near-empty log (below the threshold you choose — pick a small byte/line threshold and pin it in a test).
  - Classifier returns the non-override result (build produced output) for a real non-empty build log with normal MSBuild content.
  - `Format-BuildQueueBanner` given a synthetic result object with `-BuildFidelity 'no-output'` + `-ExitCode 1` renders `RESULT=FAIL` **and** the new next-action text.
  - Regression: `Format-BuildQueueBanner` for a normal PASS (`-ExitCode 0 -BuildFidelity 'verified'`) is unchanged.
- **Implementation goal:** A pure classifier (e.g. `Test-BuildProducedNoOutput -LogText <s>` returning `$true`/`$false`) with an explicit near-empty threshold; the banner gains one `elseif ($BuildFidelity -eq 'no-output')` arm in the next-action block. No change to the PASS path.
- **Spec requirements:** SPEC Proven Finding 2; Open Question resolutions (empty-log default detection; reuse `no-output`).
- **Batch:** 1.
