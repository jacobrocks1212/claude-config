### Phase: P1 — Per-project quarantine sweep (Root Cause A, the P0)

**Goal:** Extract one shared per-project `<root>/**/{bin,obj}` DLL enumerator and retarget both `Remove-PoisonedArtifacts` and `Get-DllLockers` onto it, so a poisoned per-project artifact (e.g. 0-byte `Cognito/bin/Debug/netstandard2.0/Cognito.dll`) is quarantined instead of surviving the worktree-root-only sweep.

**Entry criteria:** None (first phase).

**SPEC.md references:** Proven Finding 1; Root cause A (`Evidence → Source Code`); Theory 1 (Confirmed); Open Question "DLL locker sweep parity."

**Verified reality (grounding — do not re-discover):** `Remove-PoisonedArtifacts` (`build-queue-hygiene.ps1:576`) builds roots as `Join-Path $WorktreeRoot 'bin'` + `'obj'` (lines 622-625), recurses inside those two dirs only (632), poison-tests 0-byte-OR-first-2-bytes-not-`MZ` (646-663), and fail-open deletes per file via `Get-SafeValue` (669+). `Get-DllLockers` (`:898`) enumerates `Join-Path $WorktreeRoot 'bin'` **recursively** then filters `Where-Object {$_.FullName -match '[\\/]bin[\\/]Debug[\\/]'}` (953-964) — so it too is worktree-root-blind (walks `<root>/bin`, never per-project `<root>/**/bin`). Both must widen to per-project.

#### WU-1 — Shared per-project DLL enumerator + retarget both consumers

- **Scope (PHASES deliverables 1–3 + tests):**
  - [ ] New parameterized helper in `user/scripts/build-queue-hygiene.ps1` recursively enumerating `*.dll` under the worktree root across **every project subdir** (`<root>/**/bin` + `<root>/**/obj`), accepting an optional path-segment filter (e.g. `bin/Debug`) so callers restrict scope. Fail-open enumeration via the repo idiom `Get-SafeValue { ... } @()` [VERIFY: `grep -n "function Get-SafeValue" user/scripts/build-queue-hygiene.ps1`].
  - [ ] Rewrite `Remove-PoisonedArtifacts` (576-683) to call the helper with **no** path filter — sweeping both bin and obj across all project subdirs and all configs. **Preserve exactly**: the 0-byte-OR-non-`MZ` poison test (646-663) and the fail-open per-file `Get-SafeValue` delete (669+) [VERIFY: `grep -n "function Remove-PoisonedArtifacts" user/scripts/build-queue-hygiene.ps1`].
  - [ ] Retarget `Get-DllLockers` (898-~1010; enum 953-964) onto the same helper with the `bin/Debug` path filter, so both functions share one enumeration and the docstring's "mirroring" claim becomes true [VERIFY: `grep -n "function Get-DllLockers" user/scripts/build-queue-hygiene.ps1`].
- **TDD:** yes.
- **Files to create/modify:** `user/scripts/build-queue-hygiene.ps1` (add helper; rewrite two functions).
- **Test files:** `user/scripts/build-queue-hygiene.Tests.ps1` (extend).
- **Test expectations (write these It-blocks first, watch them FAIL against the current worktree-root-only sweep):**
  - Seed a 0-byte DLL under `<worktree>/Cognito/bin/Debug/netstandard2.0/Cognito.dll` → assert `Remove-PoisonedArtifacts` quarantines it (this It-block FAILS pre-fix — it is the regression pin).
  - Seed a truncated (non-`MZ`, e.g. 4 bytes of `0x00`) DLL under `<worktree>/Cognito.Core/obj/Debug/` → assert quarantined.
  - Seed a valid-PE DLL (2-byte `MZ` header + padding) under a project subdir → assert left alone.
  - `Get-DllLockers` detects a locker under a project subdir (or, if RM APIs are hard to fixture, assert the enumeration reaches a per-project `bin/Debug` DLL path — test the enumerator directly with the filter).
  - Scoping/divergence pin: a DLL under `Cognito/bin/Release/` or a non-bin/obj path is handled per each function's own filter (swept by `Remove-PoisonedArtifacts` no-filter, NOT matched by `Get-DllLockers`'s `bin/Debug` filter).
- **Implementation goal:** One helper (e.g. `Get-ProjectDlls -WorktreeRoot <r> [-PathSegmentFilter 'bin/Debug']`) enumerates `<root>` recursively for `*.dll` under any `bin`/`obj` segment, optionally post-filtered by a path-segment match, fail-open to `@()`. `Remove-PoisonedArtifacts` calls it with no filter; `Get-DllLockers` calls it with `bin/Debug`. Both consumers keep their downstream logic (poison test / RM session) unchanged — only the enumeration source is swapped.
- **Spec requirements:** SPEC Proven Finding 1 ("recurse into per-project `**/bin` + `**/obj`, converge on the `Get-DllLockers` `**/bin` pattern its docstring advertises, keeping the fail-open per-file delete and the 0-byte / non-`MZ` poison test").
- **Batch:** 1.

### Phase: P2 — Flush-retry helper for fidelity-bearing reads (Root Cause C)

**Goal:** Extract the proven active.lock retry pattern (3×/50ms) into a reusable, testable `Read-WithRetry` helper, then converge the runner's single-shot counts read and build-log read onto it so a not-yet-flushed trailing line no longer reads as empty.

**Entry criteria:** WU-1 landed (shares `build-queue-hygiene.ps1` + `.Tests.ps1`).

**SPEC.md references:** Proven Finding 3; Root cause C (`Evidence → Source Code`); Theory 3 (Confirmed, mechanism = flush/read-ordering race).

**Verified reality (grounding):** the active.lock retry exemplar lives at `build-queue-runner.ps1:229-241` (mirrored `build-queue.ps1:462-468`: `for ($attempt = 1; $attempt -le $maxAttempts; ...)` + `Start-Sleep -Milliseconds 50`, no sleep after the last attempt). The counts read is `build-queue-runner.ps1:179-192` (`[System.IO.File]::ReadAllText` at 183 + last-`Results:` regex 184-191, single-shot). The build-log read is inside the `if ($isBuildOp)` block at `build-queue-runner.ps1:141-144` (single-shot `ReadAllText`).

#### WU-2 — `Read-WithRetry` helper + Pester tests

- **Scope (PHASES deliverable 1 + tests):**
  - [ ] `Read-WithRetry`-style helper added to `user/scripts/build-queue-hygiene.ps1` (dot-sourced so Pester-testable), parameterized: a parse `[scriptblock]` returning the parsed payload or `$null` on failure, `-MaxAttempts` (default 3), `-DelayMs` (default 50); returns the first non-null result, or a fallback sentinel after exhausting attempts. Modeled on the active.lock loop **verbatim** (attempts 1..Max, `Start-Sleep -Milliseconds $DelayMs` between attempts, **no** sleep after the last attempt) [VERIFY: `grep -n "maxAttempts" user/scripts/build-queue-runner.ps1`].
- **TDD:** yes.
- **Files to create/modify:** `user/scripts/build-queue-hygiene.ps1` (add helper).
- **Test files:** `user/scripts/build-queue-hygiene.Tests.ps1` (new `Describe 'Read-WithRetry'`).
- **Test expectations (counter-closure scriptblocks — no filesystem timing dependency):**
  - Returns on the first non-null attempt (parse block returns a value immediately; assert single invocation).
  - Retries up to `-MaxAttempts` when the parse block always returns `$null`; assert it invokes exactly `MaxAttempts` times and returns the fallback sentinel.
  - Returns the fallback sentinel after `-MaxAttempts` exhausted (distinct value asserts fallback path).
  - Succeeds on the Nth attempt: a counter closure returns `$null` for attempts `< N` then a value on attempt `N`; assert the value is returned and the block invoked exactly `N` times.
- **Implementation goal:** Deterministic, fast helper; no real IO. Sleep only *between* attempts. A `$null` from the parse block means "not ready, retry"; any non-null (including `''` if the caller wants) means "done."
- **Spec requirements:** SPEC Proven Finding 3 ("adopt the existing 3×/50ms retry/settle pattern already used for `active.lock`").
- **Batch:** 2.
