### Phase: adhoc-incident-hook-deny-057921 P1 — Scope tripwire to commit pathspec

**Goal:** The `second-feature-commit` tripwire in `lazy-cycle-containment.sh` evaluates only the paths the pending `git commit` will actually include (falling back to the whole index for bare/`-a` commits and any parse ambiguity), so a pathspec-scoped commit is no longer false-denied when a concurrent lane has foreign paths staged in the shared index.

**Entry criteria:** None (first and only phase).

**TDD:** yes

**Files to create/modify:**
- `user/hooks/lazy-cycle-containment.sh` — add two module-level helpers and wire them into the tripwire.
  - Add `_commit_pathspecs(command)`: returns the explicit pathspec token list from a `git commit …` invocation, or `None` when the commit is index-wide. Return `None` (⇒ whole-index fallback) when `-a`/`--all` is present, when no explicit pathspec token follows, or on any parse ambiguity. MUST skip argument-consuming options so their values are never mistaken for pathspecs — at minimum `-m`/`--message`, `-F`/`--file`, `-C`/`--reuse-message`, `-c`/`--reedit-message`, `--author`, `--date`, `-t`/`--template`, `--fixup`, `--squash`, `--cleanup`, `-S`/`--gpg-sign` — in BOTH `--opt value` and `--opt=value` forms — and treat `--` as the explicit pathspec separator (everything after it is pathspec).
  - Add `_commit_effective_paths(command, staged)`: return `staged` unchanged when `_commit_pathspecs(command)` is `None`; otherwise return `[p for p in staged if <p matches some pathspec>]`, where a match is exact path equality OR `p` under a pathspec directory prefix (`pathspec.rstrip('/') + '/'`), path-separator-normalized (`\\`→`/`) exactly as `_is_carve_out` normalizes [VERIFY: grep -n "_is_carve_out" user/hooks/lazy-cycle-containment.sh].
  - Wire it in: at the tripwire, compute the staged set as `_commit_effective_paths(command, _staged_paths())` so the `offending` comprehension and the `_batch_docs_writer` exemption read the effective set [VERIFY: grep -n "staged = _staged_paths()" user/hooks/lazy-cycle-containment.sh]. Do NOT change `_deny(...)`, `_is_carve_out`, `_FEATURE_DIR_RE` [VERIFY: grep -n "_FEATURE_DIR_RE" user/hooks/lazy-cycle-containment.sh], `COMMIT_CEILING`, or the commit-count backstop.
