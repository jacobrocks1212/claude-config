# Ad-hoc bug: Align the unexpected-commits count numerator with the budget denominator's commit population

The process-friction unexpected-commits detector compares an authored-commit COUNT (git rev-list --count --no-merges begin..HEAD) against a per-cycle commit BUDGET, but the two model DIFFERENT commit populations, so a clean cycle false-positives whenever it makes a legitimate commit that the budget's model omits. This has now recurred twice as point-patches to the SAME structural gap: harden Round 42 (2026-06-29, algorithmic-fill-buffer) patched the NUMERATOR by excluding merge commits (--no-merges) after a sibling PR merge inflated the count; harden Round 46 (2026-06-30, audio-engine-clippy-warnings-fail-rust-gate) patched the DENOMINATOR by adding a +2 bookend term after the two deterministic status-flip commits (mark plan In-progress at start, reconcile mark plan Complete at end) plus an in-cycle revert pushed a clean 7-commit execute-plan cycle over budget 6. Each fix addressed one commit category on one side; the class is that the budget is a model of per-WU/phase AUTHORED WORK commits while the count is ALL authored non-merge commits in the range, which additionally includes bookend status-flips, in-cycle reverts/self-corrections, and unrelated off-plan commits that land on the work branch during the cycle window. Class boundary IN scope: make the count numerator and the budget denominator measure the same commit population for the unexpected-commits signal, e.g. scope the count to the cycle's own work commits (by feature-path or by cycle-window authorship) OR declare the full deterministic per-cycle commit cadence (WU commits + the 2 bookend flips + a small revert/self-correction allowance) once in a single model both sides derive from. OUT of scope: the friction-detection threshold semantics, the runaway ceiling KIND, the budget MEMBERSHIP/MAGNITUDE derivation (which is the separate adhoc-derive-multi-commit-budget-from-dispatch-sites spin-off). Cited instances: Round 42 commit b91b268 (merge exclusion, numerator); Round 46 commit 9bd61f7 (bookend term, denominator); both in lazy_core.py cycle_end_friction_check / _execute_plan_commit_budget / _count_authored_commits_since. Origin: harden-harness Round 46 (2026-06-30), which is the 2nd occurrence Round 42 explicitly predicted would warrant this spin-off.
