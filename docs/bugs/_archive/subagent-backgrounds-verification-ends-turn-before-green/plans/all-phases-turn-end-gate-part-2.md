#### Canonical replacement content (all four WUs apply this — byte-identical except the build/run verb)

The four files' step "4." is currently near-identical [VERIFY: grep -n "run_in_background: true" repos/cognito-forms/.claude/skills/msbuild/SKILL.md] — it differs only in "If the **build** …" (msbuild/nxbuild) vs "If the **run** …" (mstest/nxtest). Each WU replaces its file's step "4." with the following canonical text (substitute `<verb>` = `build` for msbuild/nxbuild, `run` for mstest/nxtest):

> 4. **If the `<verb>` is expected to exceed 10 minutes, run the same command with `run_in_background: true`** so the Bash tool returns immediately on the `build-queue: enqueued as seq=N` line. You have NOT seen the outcome yet — a backgrounded enqueue never prints the `RESULT=` banner. Before you rely on the outcome (and before ending your turn), run the followable-wait helper against that seq and trust ITS output as authoritative:
>
>    ```
>    pwsh -NoProfile -File "$HOME/.claude/scripts/build-queue-await.ps1" -Seq <N> -Op <op>
>    ```
>
>    (`<N>` is the seq from the `build-queue: enqueued as seq=N` line; `<op>` is this skill's op — e.g. `msbuild`/`mstest`/`nxbuild`/`nxtest`.) The helper blocks until `results/<seq>.json` is present, then re-emits the exact same authoritative `RESULT=` banner as its LAST stdout line and exits with the build's `exit_code`. **Trust that banner line and the exit code — do NOT `cat`/`grep` `results/<seq>.json` yourself** (the helper reads it for you; this keeps step 3's "don't read the results file" rule intact). A distinct non-zero exit with `result not yet present for seq=N` means the result is not ready — keep waiting / investigate; never treat it as success.
>
>    **Gap 2 — foreground build killed by the 10-min Bash timeout:** if you ran the command in the *foreground* and the Bash tool timed out (~10 min) BEFORE the `RESULT=` banner printed, the wrapper was killed mid-tail but the detached runner still writes `results/<seq>.json`. Recover with the SAME helper — `pwsh -NoProfile -File "$HOME/.claude/scripts/build-queue-await.ps1" -Seq <N> -Op <op>` using the seq from the `enqueued as seq=N` line — to obtain the authoritative banner. Do not conclude the build's outcome from a truncated/timed-out foreground run.
