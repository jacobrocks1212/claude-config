  - `- [ ] user/scripts/build-queue-await.ps1 (net-new)`
  - `- [ ] user/scripts/build-queue-await.Tests.ps1 (net-new Pester)`
- **TDD:** yes. Write the Pester spec first (RED), then implement to GREEN.
- **Files to create/modify:**
  - `user/scripts/build-queue-await.ps1` — **net-new (create)**
  - `user/scripts/build-queue-await.Tests.ps1` — **net-new (create)**
- **Test files:** `user/scripts/build-queue-await.Tests.ps1`
- **Test expectations (write these RED first):**
  1. **PASS-shape banner parity:** Write a fixture `results/<seq>.json` under a temp `StateRoot` with `exit_code = 0`, `counts = @{ passed = 12; failed = 0; total = 12 }`, `hygiene = @{ result_fidelity = 'verified'; build_fidelity = 'verified' }`. Run `build-queue-await.ps1 -Seq <n> -Op mstest -StateRoot <fixture-root>`. Assert the **last stdout line** equals the string produced by calling the **real** `Format-BuildQueueBanner` directly with the same field values (`-Seq`, `-Op mstest`, `-ExitCode 0`, `-ResultFidelity verified`, `-BuildFidelity verified`, `-Counts @{passed=12;failed=0;total=12}`). This crosses the real filesystem→banner boundary — do NOT mock the composer.
  2. **FAIL-shape banner parity + exit code:** Fixture with `exit_code = 1`, `counts = @{ passed = 3; failed = 2; total = 5 }`. Assert last stdout line matches the real banner for that shape AND the script's exit code equals the fixture's `exit_code` (1).
  3. **NO-TESTS-MATCHED shape:** Fixture with `exit_code = 5`, `hygiene.result_fidelity = 'no-tests-matched'`, `counts = @{ passed = 0; failed = 0; total = 0 }`. Assert the banner reflects the NO-TESTS-MATCHED result and exit code mirrors the fixture.
  4. **Await-timeout (results file absent):** Run against a `StateRoot` with NO `results/<seq>.json`, a short `-TimeoutSeconds`. Assert a **distinct non-zero exit code** (NOT the build's — pick a sentinel such as 124, and assert it is not one of the build exit codes used in cases 1–3) AND stdout/stderr contains `result not yet present for seq=<n>`.
  5. **Static source guard (sibling convention):** `Get-Content -Raw` the script and `Should -Match` that the hygiene dot-source is a top-level statement (mirrors `build-queue-hygiene.Tests.ps1:973` guard) — i.e. the dot-source is NOT nested inside a `Get-SafeValue { … }` argument.
- **Implementation goal:** A standalone script that:
  - Declares `param([Parameter(Mandatory)][int]$Seq, [string]$Op, [int]$TimeoutSeconds = 900, [int]$PollIntervalMs = 500, [string]$StateRoot = (Join-Path $HOME '.claude/state/build-queue'))`. **`-Op` is a caller-supplied parameter** because the runner does NOT persist an `op` field in `results/<seq>.json` [VERIFY: grep -n "result_fidelity" user/scripts/build-queue-runner.ps1] — the caller (the queue skill) knows the op and passes it.
  - Resolves the results file at `Join-Path $StateRoot "results/$Seq.json"`.
  - Bounded-polls for the file's presence, honoring `-PollIntervalMs` and `-TimeoutSeconds`.
  - On present: reads `exit_code`, `counts`, `hygiene.result_fidelity`, `hygiene.build_fidelity` with defensive reads via `Get-SafeValue` (any field may be missing/`$null`); dot-sources `build-queue-hygiene.ps1` **at script scope** to obtain `Format-BuildQueueBanner`; calls it with the read fields plus the caller-supplied `-Op`; writes the composed banner as the **last** stdout line; exits with the build's `exit_code`.
  - On timeout: writes `result not yet present for seq=<Seq>` and exits with a distinct sentinel non-zero code (e.g. 124), never the build's code, so the caller keeps waiting / investigates rather than treating absence as success.
  - **CRITICAL (verified gotcha):** dot-source `build-queue-hygiene.ps1` at **script scope** using the sibling pattern `. (Join-Path $PSScriptRoot 'build-queue-hygiene.ps1')` wrapped in a top-level `try { } catch { }` [VERIFY: grep -n "Join-Path \$PSScriptRoot 'build-queue-hygiene.ps1'" user/scripts/build-queue.ps1 — confirmed at build-queue.ps1:48]. Do **NOT** place the dot-source inside a `Get-SafeValue { … }` argument — `Get-SafeValue` runs its block in a child scope (`& $Block`), so any functions defined by the dot-source would be discarded and `Format-BuildQueueBanner` would be undefined. The Tests.ps1 guard (test case 5) enforces this.
  - Reuse the ubiquitous `Get-SafeValue` helper [VERIFY: grep -n "function Get-SafeValue" user/scripts/build-queue-hygiene.ps1 — confirmed at :41]. Since the script dot-sources `build-queue-hygiene.ps1` at script scope, `Get-SafeValue` is in scope after the dot-source — but define a local copy near the top (matching the sibling convention where each build-queue script defines its own copy) so defensive reads work even before/independent of the dot-source.
  - Repo PowerShell conventions: `$null` never `/dev/null`; defensive reads matching `build-queue-hygiene.ps1` / `build-queue-runner.ps1`.
- **Spec requirements:**
  - `Format-BuildQueueBanner` is pure/side-effect-free/non-throwing with params `-Seq [int]`, `-Op [string]`, `-ExitCode [int]`, `-ResultFidelity [string]`, `-BuildFidelity [string]`, `-Counts [hashtable]` [VERIFY: grep -n "function Format-BuildQueueBanner" user/scripts/build-queue-hygiene.ps1 — confirmed at :1419]. Note: the fn reads only the `total` and `failed` keys of `-Counts` (not `passed`); pass all three regardless, matching the wrapper's `$bannerCounts` at `build-queue.ps1:489-493`.
  - The runner writes `results/<seq>.json` as `[ordered]@{ seq; exit_code; ended_at; counts; hygiene = @{ vbcscompiler_recycled; recycle_skipped_reason; quarantined_artifacts; result_fidelity; build_fidelity; lockers_reaped } }` — top-level `exit_code`/`counts`, nested `hygiene.{result_fidelity,build_fidelity}`, **no `op`** [VERIFY: grep -n "result_fidelity" user/scripts/build-queue-runner.ps1].
  - The Pester spec matches sibling conventions in `user/scripts/build-queue-hygiene.Tests.ps1` [VERIFY: test -f user/scripts/build-queue-hygiene.Tests.ps1]: Pester v5, a top-level `BeforeAll { $script:ModulePath = Join-Path $PSScriptRoot 'build-queue-await.ps1'; … }`, `Describe`/`It` blocks, `Should -Be`/`-Match`/`-Not -Throw`, `Get-Content -Raw` source-regex guards. No `Mock`/`InModuleScope`/`Should -Invoke`. The fixture `results/<seq>.json` lives in a per-test temp dir (`$TestDrive` is acceptable here even though the sibling doesn't use it, because this test needs a real filesystem fixture — or use `[System.IO.Path]::GetTempPath()` + a Guid subdir with `AfterEach` cleanup; either is fine as long as it is a real on-disk path).
