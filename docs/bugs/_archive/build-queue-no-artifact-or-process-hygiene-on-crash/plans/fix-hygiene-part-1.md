### Phase: P1 — Shared hygiene module + Job-Object descendant reaping

**Goal:** Create the dot-sourced hygiene module and reap the build's descendant process tree via a Windows Job Object on every exit path (normal, non-zero, runner abort/kill).
**Entry criteria:** None (first phase; establishes the shared module).
**SPEC.md references:** Proven Findings 1–4; Locked Decision 2 (Windows Job Object scoping — not a parent-PID tree walk); Affected Area rows "Detached build runner" + "Stale-reclaim / liveness".

#### WU-1 — Hygiene module + Job-Object reap helpers

- **Scope (PHASES P1 deliverables 1, 3, 4):**
  - [x] Create `user/scripts/build-queue-hygiene.ps1`, dot-sourceable, exposing the Job-Object P/Invoke surface: `New-BuildJobObject` (`CreateJobObject` + `SetInformationJobObject` with `JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE`), `Add-ProcessToBuildJob` (`AssignProcessToJobObject`), `Stop-BuildJobTree` (`TerminateJobObject` + close handle).
  - [x] Every public function FAILS OPEN — a non-Windows host or any P/Invoke failure logs a warning (to stderr/`Write-Warning`) and returns a benign sentinel; it MUST NOT throw in a way that aborts the caller's build.
  - [x] The reap scopes to Job-Object membership ONLY — there is no `Get-Process dotnet | Stop-Process` global kill anywhere in the module (Locked Decision 2: never touches a sibling worktree's live build).
  - [x] Optional Pester smoke (`user/scripts/build-queue-hygiene.Tests.ps1`) for the non-P/Invoke surface (fail-open guards, parameter validation, the no-global-kill property).
- **TDD:** partial — the P/Invoke create/assign/terminate cannot be unit-tested (needs real OS handles + processes; covered by PHASES Runtime Verification). The fail-open guards and the "no global Stop-Process" invariant ARE Pester-testable.
- **Files to create/modify:**
  - `user/scripts/build-queue-hygiene.ps1` — **net-new** module.
  - `user/scripts/build-queue-hygiene.Tests.ps1` — **net-new** Pester smoke.
- **Test files:** `user/scripts/build-queue-hygiene.Tests.ps1`.
- **Test expectations (Pester):**
  - `New-BuildJobObject` / `Add-ProcessToBuildJob` / `Stop-BuildJobTree` are dot-sourceable and defined (function existence).
  - Calling a public function on a simulated P/Invoke failure path returns the benign sentinel and does NOT throw.
  - A static scan of the module source asserts NO `Stop-Process` targeting a process-name glob (the no-global-kill guard) — grep the module text inside the test.
- **Implementation goal:** A standalone, dot-sourceable PowerShell module owning the Job-Object lifecycle so the runner (WU-2) can scope and reap exactly one build's descendant tree. Use the `Add-Type`/`[DllImport]` P/Invoke pattern for `kernel32.dll` (`CreateJobObjectW`, `SetInformationJobObject`, `AssignProcessToJobObject`, `TerminateJobObject`, `CloseHandle`). Reuse the `Get-SafeValue { … } $fallback` fail-open idiom already used in `build-queue-runner.ps1` [VERIFY: grep -n "function Get-SafeValue" user/scripts/build-queue-runner.ps1] (defined at runner.ps1:34) and `build-queue.ps1` [VERIFY: grep -n "function Get-SafeValue" user/scripts/build-queue.ps1] — copy/define the same helper in the module rather than inventing a new error idiom.
- **Spec requirements:** Locked Decision 2 ("Windows Job Object — assign the runner's build grandchild to a Job Object at launch; reap by terminating the job. Reliably scopes to exactly this build's tree and survives PID reuse; never kills a sibling worktree's live build. Not the parent-PID tree walk."); Proposed Fix Direction "Reap build descendants on completion and on abort."

### Phase: P2 — Force-recycle VBCSCompiler between queued builds

**Goal:** Shut down the machine-global `VBCSCompiler` server after each queued build (or abort) so the next build cold-starts a fresh compiler server instead of reusing a half-dead node (the cross-worktree `MSB4166` vector).
**Entry criteria:** P1 complete — reuses the runner's `try/finally` bracket (WU-2) and the `build-queue-hygiene.ps1` module (WU-1).
**SPEC.md references:** Locked Decision 1 (force-recycle every build, not reap-only-when-unhealthy); Cross-worktree vector section (machine-global `VBCSCompiler` / MSBuild node reuse → `MSB4166`); Proposed Fix Direction "Force-recycle VBCSCompiler between queued builds."
