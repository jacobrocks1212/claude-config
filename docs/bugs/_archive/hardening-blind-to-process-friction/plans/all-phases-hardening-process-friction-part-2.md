### Phase: hardening-blind-to-process-friction P2 — Process-friction detector → deny ledger

**Goal:** Make a torn cycle bracket or unexpected commits append a `kind: process-friction` entry to `lazy-deny-ledger.jsonl` so the existing probe withholding + `--run-end` refusal fire exactly like a guard deny — the runaway self-announces as hardening debt.

**Entry criteria:** Phase 1 complete (part 1 landed; the cycle marker is retained and any subagent-context mechanism is established).

**SPEC.md references:** Locked Decisions **D1** (detector scope: cycle-bracket break + unexpected commits), **D1-out** (runtime-death heuristics deferred), **D3** (fix detection + prevention); Affected Area "Detector → debt ledger"; resolved Open Questions (run-identity = `started_at`; HEAD-SHA snapshot; ledger reuse).

#### WU-2 — `detect_cycle_bracket_friction` pure detector (D1 a+b)

- **Scope:** PHASES Phase 2 deliverable 3.
- **TDD:** yes.
- **Files to create/modify:** `user/scripts/lazy_core.py` (new `detect_cycle_bracket_friction`).
- **Test files:** `user/scripts/test_lazy_core.py`.
- **Test expectations:**
  - clean bracket (run identity unchanged, commits within budget) → returns None.
  - torn bracket (marker's `run_started_at` differs from the live run marker's `started_at`, OR the run marker is now absent when it was present at begin) → returns a friction descriptor with reason "cycle-bracket-break".
  - over-budget commits (HEAD advanced beyond the conservative per-`sub_skill` budget from `begin_head_sha`) → returns a descriptor with reason "unexpected-commits".
  - degraded inputs (null `run_started_at`/`begin_head_sha`) → returns None (never a false positive), no crash.
- **Implementation goal:** `detect_cycle_bracket_friction(marker, current_run_started_at, current_head_sha, sub_skill, *, now=None)` — pure function implementing D1(a) bracket-break and D1(b) unexpected-commits, returning a friction descriptor (`{reason, detail, ...}`) or None. The expected-commit budget per `sub_skill` is conservative (most sub-skills 0–1 commits); pick a defensible default (commits > 1 beyond `begin_head_sha`, OR any bracket break, trips it) and document the budget map inline. No I/O — caller supplies the live values.
- **Spec requirements:** Locked Decision D1 (both deterministic on-disk signals); D1-out (no runtime-death heuristic).
- **Batch:** 2.

#### WU-3 — `append_friction_ledger_entry` + `build_hardening_emit_command` binding

- **Scope:** PHASES Phase 2 deliverables 4 + 6.
- **TDD:** yes.
- **Files to create/modify:** `user/scripts/lazy_core.py` (new `append_friction_ledger_entry`; extend `build_hardening_emit_command` `:6332` [VERIFY: `grep -n "def build_hardening_emit_command" user/scripts/lazy_core.py`] + `oldest_unacked_deny`).
- **Test files:** `user/scripts/test_lazy_core.py`.
- **Test expectations:** `append_friction_ledger_entry` appends a `kind: "process-friction"`, `acked: false` line to the SAME `lazy-deny-ledger.jsonl` and `pending_hardening()` then returns ≥1 [VERIFY: `grep -n "def append_deny_ledger_entry" user/scripts/lazy_core.py`]; `build_hardening_emit_command` given a process-friction oldest-entry emits `--context trigger_kind=process-friction` and surfaces the friction detail (NOT `denied_prompt_summary`/`denial_reason`).
- **Implementation goal:** `append_friction_ledger_entry(reason_head, detail, now=None)` mirrors `append_deny_ledger_entry`'s shape + fail-open contract but writes `kind: "process-friction"` so a single reader walks denies + friction. `build_hardening_emit_command` detects a process-friction entry (via its `kind`) and binds `trigger_kind=process-friction` + a friction-detail `--context` instead of the deny-specific bindings. Keep `pending_hardening`/`oldest_unacked_deny`/`--run-end` gate consumption unchanged (they already count any unacked entry).
- **Spec requirements:** Affected Area "Detector → debt ledger"; resolved Open Question (ledger reuse, not a sibling file).
