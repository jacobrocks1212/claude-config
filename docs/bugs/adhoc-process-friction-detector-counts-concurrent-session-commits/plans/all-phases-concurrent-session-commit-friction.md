#### WU-1 — concurrent-activity ledger substrate

- **Scope:** PHASES.md Phase 1 deliverables 1–2 — `append_concurrent_commit_sha(sha, *, run_started_at)` and `read_concurrent_commit_entries()`.
- **TDD:** yes
- **Files to create/modify:** `user/scripts/lazy_core/ledgers.py`
- **Test files:** `user/scripts/tests/test_lazy_core/test_ledgers.py`
- **Test expectations:**
  - `append_concurrent_commit_sha("abc123", run_started_at="R1")` writes one compact JSON line `{sha, run_started_at, ts}` to `claude_state_dir() / "lazy-concurrent-activity.jsonl"`; returns True.
  - `read_concurrent_commit_entries()` returns the appended entries as `{sha: run_started_at}` (or a list of dicts — pick one shape and assert it); a missing ledger file returns an empty result (never raises); a torn/partial final line is tolerated (the valid lines still parse).
  - Fail-open: an unwritable state dir (simulate by pointing `LAZY_STATE_DIR` at a path that cannot be created, or monkeypatch the open) makes `append_concurrent_commit_sha` return False WITHOUT raising.
  - `run_started_at=None` (interactive, no live marker) records `run_started_at: null` and round-trips.
- **Implementation goal:** Add the two helpers next to `append_deny_ledger_entry` [VERIFY: grep -n "def append_deny_ledger_entry" user/scripts/lazy_core/ledgers.py], REUSING its exact fail-open plain-append pattern (`claude_state_dir() / <FILENAME>`, `open("a")`, compact `json.dumps`, `except` → return False). Introduce a module constant `_CONCURRENT_ACTIVITY_FILENAME = "lazy-concurrent-activity.jsonl"`. Do NOT re-implement the state-dir resolution or the append idiom — mirror the deny-ledger helper. `read_concurrent_commit_entries` reads via `claude_state_dir(create=False)` (mirror `read_deny_ledger` [VERIFY: grep -n "def read_deny_ledger" user/scripts/lazy_core/ledgers.py]), tolerant of a missing file and a torn final line.
- **Spec requirements:** SPEC `## Candidate Fix Approaches` → Approach B ("append their produced commit sha(s) to a concurrent-activity ledger in the shared per-repo keyed state dir"). The per-repo keyed state dir is shared across concurrent same-repo sessions — that shared visibility is exactly what makes the operator's interactive-session commits observable to the batch run's detector.
- **Batch:** 1

#### WU-2 — extend the detector with the ledger-read arm

- **Scope:** PHASES.md Phase 1 deliverables 3–4 — extend `_count_concurrent_writer_commits` and wire the caller.
- **TDD:** yes
- **Files to create/modify:** `user/scripts/lazy_core/markers.py`
- **Test files:** `user/scripts/tests/test_lazy_core/test_markers.py`
- **Test expectations (extend the existing concurrent-writer fixtures at test_markers.py:4008 / :4046):**
  - **Same-identity subtraction:** seed `lazy-concurrent-activity.jsonl` with N in-window shas stamped `run_started_at="OTHER"`; a temp git repo whose `begin_head_sha..HEAD` contains those N commits (all under the repo's own single `user.email`); assert `_count_concurrent_writer_commits(repo, begin, current_run_started_at="MINE")` returns N (the pre-fix email-only path returns 0).
  - **Detector integration:** `detect_cycle_bracket_friction` with `commits_since = N + budget` and those N ledger-recorded concurrent commits does NOT emit `unexpected-commits` (chargeable = budget); with `commits_since = N + budget + 1` it still trips (remainder over budget).
  - **This-run shas NOT subtracted:** a ledger sha stamped `run_started_at="MINE"` (== current) is NOT counted → this run's own automated commits still charge against the budget (a genuine same-run runaway still trips).
  - **Fail-safe on degraded read:** missing/unreadable ledger degrades to the email-only count (byte-identical to today — no new suppression); an entry with absent/malformed `run_started_at` is NOT subtracted (conservative).
  - **De-dup:** a window commit that is BOTH a distinct committer email AND ledger-recorded is counted exactly once.
  - **`None` contract preserved:** a degraded git read (no begin sha / unreadable config / git failure) still returns `None` (disables suppression at the caller).
- **Implementation goal:** Widen `_count_concurrent_writer_commits` [VERIFY: grep -n "def _count_concurrent_writer_commits" user/scripts/lazy_core/markers.py] to `(repo_root, begin_head_sha, current_run_started_at)`. After the existing email-inequality sum, add a ledger arm: read `begin_head_sha..HEAD` full shas (`git log --no-merges --format=%H`), read `read_concurrent_commit_entries()` [VERIFY: grep -n "def read_concurrent_commit_entries" user/scripts/lazy_core/ledgers.py] (this WU depends on WU-1), and UNION in window shas whose ledger `run_started_at` is present AND `!= current_run_started_at`, de-duplicated against the email-attributed set. A ledger-read failure is caught and degrades to the email-only count (NEVER over-subtracts). Then wire the caller `cycle_end_friction_check` [VERIFY: grep -n "def cycle_end_friction_check" user/scripts/lazy_core/markers.py] to pass its already-computed `current_run_started_at` (the value read at markers.py:2178) into the call at markers.py:2226.
- **Spec requirements:** SPEC `## Proven Findings` — "A fix must only ADD positive same-identity attribution; it must not turn ambiguity into suppression." The ledger arm is a POSITIVE signal only; every degraded/ambiguous path falls back to the current behavior.
- **Batch:** 2 (depends on WU-1's `read_concurrent_commit_entries`).
