### Phase: hardening-blind-to-process-friction P1 — `agent_id`-targeted containment

**Goal:** Replace marker-armed recursion/lifecycle containment with `agent_id`-targeted containment so a dispatched subagent is denied recursive `/lazy-batch`/`Agent`/routing/lifecycle ops while the orchestrator is never self-denied — with NO marker arming required.

**Entry criteria:** None (first phase).

**SPEC.md references:** Locked Decision **D4** (arming-free `agent_id`-targeted prevention); Proven Findings #2 (marker never armed), #3 (orchestrator-self-deny defect); Open Question "`agent_id` field confirmation (BLOCKING)".

- **TDD:** yes.

- **Test files:** `user/scripts/test_hooks.py`.
- **Test expectations:** A fixture that invokes `user/hooks/lazy-cycle-containment.sh` (its inline Python is driven via the hook's stdin contract — feed a PreToolUse JSON payload on stdin) with:
  - subagent-shaped payload — `agent_id` present (e.g. `{"agent_id": "...", "tool_name": "Agent", "tool_input": {...}}`) → asserts the hook emits `permissionDecision: deny`.
  - main-thread-shaped payload — `agent_id` absent, same `tool_name: "Agent"` → asserts the hook emits NO deny (allow).

If the installed version exposes the subagent marker under a different field, pin THAT name in the fixture and note it. #### WU-2 — Re-target the C2 hook (`lazy-cycle-containment.sh`) on `agent_id`

- **Scope:** PHASES Phase 1 deliverable 2.
- **TDD:** yes (turns WU-1's fixtures green).
- **Files to create/modify:** `user/hooks/lazy-cycle-containment.sh` [VERIFY: `grep -n "tool_name == \"Agent\"" user/hooks/lazy-cycle-containment.sh`].
- **Test files:** covered by `user/scripts/test_hooks.py` (WU-1) — extend with one case per op class (recursive `Agent`, `/lazy-batch` invocation, a `LOOP_FORMATION_FLAGS` routing call, a `LIFECYCLE_PATTERNS` `dev:kill`), each asserting deny-when-`agent_id`-present / allow-when-absent.
- **Implementation goal:** In the hook's inline Python `main()`, gate the recursion/lifecycle/routing deny branches on `payload.get("agent_id")` (the confirmed field) being present:
  - `tool_name == "Agent"` (and `Task` if present) → deny ONLY when `agent_id` present; allow when absent.
  - `_STATE_PY_RE` + `LOOP_FORMATION_FLAGS` routing → deny ONLY when `agent_id` present.
  - `LIFECYCLE_PATTERNS` (`dev:kill`/`dev:restart`) → deny ONLY when `agent_id` present.
  - Also deny a `/lazy-batch` invocation in a Bash command when `agent_id` present.
  - **Retain** the marker-gated 2nd-feature tripwire + commit-ceiling backstop unchanged (`feature_id`/`commit_tally` still read from the marker) — those stay marker-gated; only recursion/lifecycle/routing move to `agent_id`.
  - Adjust the bash fast-path: it currently exits 0 when the marker is absent. The new trip no longer needs the marker, so the fast-path must NOT short-circuit on marker-absence for the `agent_id` branches. Resolve this so the hook still runs its Python for subagent calls even with no marker (the simplest correct shape: drop/relax the `[ ! -f "$MARKER" ]` early-exit so the inline Python always evaluates `agent_id`; keep fail-OPEN-via-empty-output and the no-python fallback). Preserve zero-overhead for the common main-thread interactive case by having the Python fast-allow immediately when `agent_id` is absent AND the marker is absent.
  - Update the hook's header comment block to describe the `agent_id` trip + retained marker-gated tripwires.
- **Spec requirements:** Locked Decision D4; Proven Finding #3 (no `agent_id` discrimination today).
- **Batch:** 2 (depends on WU-1's fixtures existing).

The marker-presence check stays as the backstop carrier, but a subagent-context signal (not mere marker-presence) drives the refusal so the orchestrator running these ops between cycles (marker cleared) is unaffected — and, critically, the orchestrator is never self-refused even if a stale marker lingers. Update the lockstep comment above `CYCLE_REFUSED_OPS` to describe the `agent_id` trip and the retained marker-gated tripwires so the hook deny-set and `CYCLE_REFUSED_OPS` stay auditable in lockstep.
