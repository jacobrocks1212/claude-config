## Phase: continuity P1 — Enumerated continuity/fresh partition SSOT + completeness assertion

**Goal:** Introduce the two enumerated partition constants in `lazy_core.py` and a test-enforced completeness invariant binding them to the `write_run_marker` literal, so a new run-scoped marker field can never silently default to the reset side.
**Entry criteria:** None (first phase).
**SPEC.md references:** Proven Findings 1–2 (the two-seam split, no enumerated allow-list); Affected Area "Marker mint" + "Regression net" rows; the SPEC's recommended durable-fix shape (Proven Finding 2 — the `RUN_CONTINUITY_FIELDS` / `RUN_FRESH_FIELDS` SSOT + `--test` completeness assertion).

### WU-1 — Partition SSOT + key-set helper

- **Scope (PHASES Phase 1 deliverables 1–3):**
  - Add `RUN_CONTINUITY_FIELDS` constant (a `frozenset`) in `lazy_core.py` near the marker-constants region — `{"forward_cycles", "meta_cycles", "started_at", "per_feature_forward_cycles", "per_feature_corrective_cycles"}`, with a docstring naming the carry-vs-reset contract + the super-invariant (run-scoped continuity survives a sanctioned same-run pause).
  - Add `RUN_FRESH_FIELDS` constant (a `frozenset`) = the remaining run-scoped keys of the `write_run_marker` literal — `{"last_advance_consume_count", "pipeline", "cloud", "repo_root", "session_id", "max_cycles", "nonce_seed", "attended", "work_branch"}`.
  - Add a helper that returns the ACTUAL run-scoped key set of a freshly-minted marker (e.g. `_run_marker_scoped_keys() -> frozenset[str]`), so the completeness assertion checks against the live literal, not a hand-copied list that could drift. Simplest correct form: call `write_run_marker` into a temp/throwaway state dir is overkill — instead derive from a minted marker dict's `.keys()` in the test, OR expose a module-level tuple the literal is built from. Implementer's choice; the REQUIREMENT is that the assertion's "actual key set" cannot drift from the real literal silently. Prefer reading the keys off a minted marker dict in the test (WU-2) over duplicating the key list in source.
- **TDD:** no (pure constant + helper additions; the behavior is asserted by WU-2's test, which is the test-first artifact for this pair — WU-1 and WU-2 are the test/impl halves of one TDD concern and may be implemented test-first as a unit).
- **Files to create/modify:** `user/scripts/lazy_core.py` [VERIFY: grep -n "_MARKER_FILENAME = " user/scripts/lazy_core.py] [VERIFY: grep -n "def write_run_marker" user/scripts/lazy_core.py] — add the two constants + helper near the constants region (`_MARKER_FILENAME` is at line ~6162; place the partition SSOT in that region with a back-reference comment to the `write_run_marker` literal it partitions).
- **Implementation goal:** The two frozensets exist as the SSOT partition; their union equals the run-scoped key set of the `write_run_marker` literal [VERIFY: grep -n "marker: dict = {" user/scripts/lazy_core.py] and they are disjoint. Do NOT change any value `write_run_marker` mints.
