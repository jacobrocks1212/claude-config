# Intervention — harden-2026-07-r67

Cycle-budget counters double-count on probes + the per-turn inject hook
(`docs/bugs/cycle-budget-counters-double-count-on-probes-and-inject-hook`). Operator
directive: offload cycle-counting to the SCRIPT so a cycle counts only on a real dispatch,
decoupled from probes. `forward_cycles` was advanced by `advance_forward_cycle` on the
`--repeat-count` probe path (fires on state-change), so inspection probes AND the per-turn
inject-hook probe inflated the budget with no dispatch. The fix relocates the increment to
the `--cycle-end` bracket (new `advance_cycle_bracket_counter`, keyed on `--kind`), keeps
`--apply-pseudo`, and removes the probe-path + emit-dispatch advances. Probes are now
budget-neutral; a real bracket counts exactly once.

**Measured effect (intended):** forward_cycles delta per inspection/inject probe == 0; per
completed real `--cycle-begin --kind real`/`--cycle-end` bracket == +1 (with the per-feature
sibling). Anchored to `event:cycle-end` (the new advance event). The durable guarantee is the
five regression tests in `test_markers.py` (a–e). Honest residual: `__flip_plan_complete_stale__`
(applied inline, unbracketed) no longer has its meta cycle counted — budget-neutral since
`meta_cycles` is uncapped/display-only.
