# Intervention — harden-2026-07-r65

Research-gated head buried by default skip-ahead + merged-view fallthrough
(`docs/bugs/research-gated-head-buried-by-skip-ahead-and-merged-fallthrough`). Round 64's
`probe_skipped_ids` folded research-pending gated heads into the merged-head exclude set
identically to BLOCKED heads, so a high-priority research-gated feature at the queue head was
silently buried behind a lower-priority bug the merged view fell through to — the operator
never saw the needs-research halt. The fix classifies gated heads by kind
(`_gated_head_kind`), surfaces `research_gated_heads`, and (via
`lazy_core.dispatch.research_halt_head`) re-emits a scoped needs-research terminal when a
research head is the research-inclusive merged head (outranks the fallthrough), so the
driver's Step-4 halt surfaces the prompt. A lower-priority research head is not surfaced
(no over-halt). Provisional design-fork (contained divergence); precedence decisions are
ratification-pending in the sibling `NEEDS_INPUT_PROVISIONAL.md`.

**Measured effect (intended):** an INCREASE in correct needs-research halts (`event:halt`)
for research-gated heads that were previously silently skipped/buried. Immeasurable in the
strict window here (CLI blocked — see baseline.reason); the durable guarantee is the
regression test `test_subprocess_emit_prompt_surfaces_needs_research_over_lower_bug`.
