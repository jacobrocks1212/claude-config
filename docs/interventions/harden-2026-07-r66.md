# Intervention — harden-2026-07-r66

Non-P0 bug outranks a P1 feature at an age-escalated tie
(`docs/bugs/non-p0-bug-outranks-p1-feature-on-aged-tie`). Operator directive: "I only want
P0 bugs to sort ahead of P1 features." An aged P2 bug (age-escalated to rank-1
P1-equivalent) beat a P1 (pre-release) feature because the equal-rank tie-break
(`_MERGED_TYPE_ORDER = {"bug": 0, "feature": 1}`) favored the bug. The fix flips the
tie-break to `{"feature": 0, "bug": 1}` (feature wins ties); combined with the rank-1
age-escalation floor (no aged bug reaches rank 0) this yields EXACTLY "only a genuine P0
bug precedes a P1 feature".

**Measured effect (intended):** the ordering invariant — a non-P0 bug never precedes a P1
feature in the merged worklist — holds. No ledger event tracks merged ordering
(`target_signal: undeclared`); the durable guarantee is the regression test
`test_merged_worklist_aged_p2_bug_sorts_behind_p1_feature` (+ the retained
`test_merged_worklist_bug_breaks_tie_at_equal_priority`).
