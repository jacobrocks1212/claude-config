# Intervention: build-queue nxbuild false-no-output fix

**Hardening round:** Round 121, 2026-07-17, `build-queue-nxbuild-false-no-output-negative`

**Defect:** build-queue-runner.ps1 Read-WithRetry exhausts on slow Nx I/O (>1s to settle), leaving `$buildLogTextForClassify` null, causing the no-output classifier to falsely fail successful builds.

**Intervention:** Added extended fallback read (15x/100ms = 1.5s additional) after initial retry (10x/100ms = 1s) completes, ensuring classifier gets access to log text if file becomes available after initial window. Commit: `a291079`.

**Signal tracking:** The no-output classification defect is self-observed (build log read exhaustion) and not directly emitted as a telemetry-ledger event. A later run detecting the defect would be observable indirectly via `build_fidelity='no-output'` in results JSON, but there is no unified KPI or deterministic recurrence marker in the lazy pipeline. This intervention cannot be graded as measured/unmeasured without instrumenting the build-queue runners to emit a "read-exhaustion" event class.

**Verdict:** INCONCLUSIVE — the intervention is mechanically sound (extended fallback read ensures log text capture), but the recurrence-frequency baseline cannot be computed from existing signals. The defect is confirmed by manual observation (seq=1239); future occurrences would be confirmed the same way (via `/build-queue-status` + manual build log inspection) until a dedicated event is added.

**Status:** complete, ready for ship.
