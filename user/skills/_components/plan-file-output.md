## Plan File Output (Mobile Workflow)

This skill writes the plan to a Markdown file instead of entering plan mode. This enables a mobile workflow where plans are generated remotely and then executed by directing Claude Code to the plan file in a separate session.

### Output Path Convention

1. **If a PHASES.md was resolved:** write the plan as `PLAN.md` in the same directory as the PHASES.md (e.g., `docs/features/audio/.../PLAN.md`)
2. **If standalone (no PHASES.md):** write the plan to `docs/plans/PLAN-{slug}.md` where `{slug}` is a kebab-case summary of the work (e.g., `docs/plans/PLAN-fix-crossfade-glitch.md`). Create `docs/plans/` if it doesn't exist.

### Write Protocol

1. **Write the plan** using the `Write` tool to the resolved path. The plan content is identical to what would have been presented in plan mode — fully self-contained, all sections included.
2. **Add a preamble** at the very top of the plan file:

```
> **Mobile plan** — generated by `/{skill-name}` on {date}.
> To execute: open a new Claude Code session and say "Execute the plan at `{path}`"
> This plan is fully self-contained. The executing session needs no additional context.
```

3. **Report to user:** Print the plan file path so it can be referenced from the mobile app:

```
Plan written to: {absolute-path}
Execute it by telling Claude Code: "Execute the plan at {path}"
```

### IMPORTANT: No Plan Mode

- Do NOT call `EnterPlanMode` or `ExitPlanMode`
- Do NOT present the plan for interactive approval
- The plan file IS the deliverable of this skill invocation
