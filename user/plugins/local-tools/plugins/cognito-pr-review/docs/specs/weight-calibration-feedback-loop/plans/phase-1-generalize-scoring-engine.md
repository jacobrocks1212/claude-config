  The test agent MUST first `Read scripts/post-process.ts` in full to capture the exact `CombinedFindings` input shape, the `Manifest` shape, the `WeightsConfig` shape (`source_weights` will be present after WU-1), and the CLI contract. Verified facts to build on:
  - The CLI is `npx tsx post-process.ts --input <file> --manifest <file>` and the result JSON is written to **stdout** (`process.stdout.write(...)`, `main()` runs unconditionally) — so test black-box via `execFileSync("npx", ["tsx", "post-process.ts", "--input", fx, "--manifest", mf], { cwd: scriptsDir })` and `JSON.parse` the captured stdout. Do NOT import internals (that would execute `main()`).
  - `step5_filterOutOfScope` drops findings whose `file` is not in the manifest — so EVERY fixture finding's `file` MUST be listed in `phase1-manifest.json`, or the weight assertions are masked by scope-filtering. This is the #1 fixture gotcha.

  Fixture `phase1-combined-findings.json` must contain at least:
  1. one `investigation` finding (no `confidence` field) on file `A`,
  2. one `reuse` finding (no `confidence`) on file `B`,
  3. one `intrafile` finding (no `confidence`) on file `C`,
  4. one `sweep` finding with a known `rule_id`/`rule_category` present in `weights.yaml` on file `D`,
  5. one `investigation` finding carrying `confidence: "UNVERIFIED"` on file `E`,
  6. one `intrafile` finding carrying `confidence: "UNVERIFIED"` on file `F` (so `0.7 × 0.5 = 0.35`, survives) AND one with a source-weight low enough OR an explicit low confidence to land **below** `0.3` so a drop is observable — pick values from the real `weights.yaml` after reading it (e.g. an `intrafile`/`reuse` finding with `confidence: "UNVERIFIED"` where the source weight makes `weight × 0.5 < 0.3`; if `0.7 × 0.5 = 0.35` never drops, add a fixture finding whose source weight is exercised via a deliberately low value — see note),
  7. an Opus-source finding and a `sweep` finding on the **same** `file:line` (to assert dedup precedence).

  > **Drop-observability note:** with defaults `0.9/0.7/0.7` and `UNVERIFIED=0.5`, the lowest non-sweep effective weight is `0.7 × 0.5 = 0.35`, which does NOT drop at the `0.3` floor. To assert a non-sweep drop deterministically, the test should drive a fixture where the dropped finding is a **sweep** finding whose `rule_weight × category_multiplier × confidence < 0.3` (e.g. a `0.525` rule in the `consistency` category `× 0.8 × UNVERIFIED 0.5 = 0.21`), OR add a temporary low `source_weights` value via a dedicated fixture-scoped weights path if the agent finds it cleaner. The REQUIRED assertion is: "a finding below `0.3` after `weight × confidence` is dropped, regardless of source." Choose fixture numbers (read from `weights.yaml`) that make at least one non-sweep-or-sweep finding cross the floor.

  Assertions:
  - **A1:** the `investigation` finding (no confidence) has `effective_weight === 0.9` (was `1.0` before) in `processed_findings`.
  - **A2:** the `reuse` and `intrafile` findings (no confidence) have `effective_weight === 0.7`.
  - **A3:** the `sweep` finding's `effective_weight === rule_weight × category_multiplier × 1.0` (unchanged formula, now also multiplied by the defaulted confidence).
  - **A4:** the `UNVERIFIED` investigation finding has `effective_weight === 0.9 × 0.5 === 0.45`.
  - **A5:** a finding whose `weight × confidence < 0.3` is ABSENT from `processed_findings` and `dropped_count` increments accordingly (the drop now applies to non-sweep sources too — the `source === "sweep"` guard is gone).
  - **A6:** on the colliding `file:line`, the surviving finding's `source` is the Opus-lane one (investigation/reuse/intrafile), not sweep — ranking/dedup precedence preserved.
- **Implementation goal:**
  - Add a `resolveConfidence(finding): number` helper that maps a finding's `confidence` label to a number: `"CONFIRMED" → 1.0`, `"UNVERIFIED" → 0.5`, **absent/unknown → 1.0** (back-compat: Phase 2 makes agents emit the label; until then everything defaults to `1.0` so behavior is preserved). Put the label→number mapping HERE in the engine — Phase 2 agents emit only the string. [VERIFY anchor for the field name: agents do not yet emit `confidence`; this is forward-compatible.]
  - Generalize `computeEffectiveWeight()` (`:218`): for `source === "sweep"`, keep `rule_weight × category_multiplier`; for `investigation`/`intrafile`/`reuse`, use `weights.source_weights[source]` (default `0.7` if a key is somehow missing). Then multiply the result by `resolveConfidence(finding)` for ALL sources.
  - Replace the hardcoded `effective_weight: 1.0` at the investigation/reuse/intrafile wrap sites (`:320/:346/:372`) with the generalized `computeEffectiveWeight(...)` call.
  - Generalize `step2_dropBelowThreshold` (`:391`): remove the `f.source === "sweep"` condition so the `MIN_EFFECTIVE_WEIGHT` (`0.3`, unchanged) floor applies to all sources.
  - **Preserve** `step3_deduplicate` Opus-over-sweep preference (`:427`) and `step4_rank` (Opus forced to "critical" tier) exactly — do not let the weight generalization change source precedence. The dedup tie-break may still use `effective_weight`, but Opus-vs-sweep must resolve to Opus first.
  - Do NOT change `MIN_EFFECTIVE_WEIGHT`, `loadWeights()`, `getCategoryMultiplier()`, the CLI args, or the stdout output shape.
- **Spec requirements:** SPEC Fix Direction R1; Proven Finding #1; PHASES.md Validated Decisions (discrete confidence mapping, floor stays `0.3`).
