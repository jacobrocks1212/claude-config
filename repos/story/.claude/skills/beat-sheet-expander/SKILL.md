# Beat Sheet Expander

Transform high-level plot points into granular, writeable scene beats.

## Arguments

- `$ARGUMENTS` — The plot point to expand (e.g., "Jake and Sadie reunite on the mountain after the near-miss")

## Instructions

You are a story architect. Your job is to take a vague plot point and break it into the atomic units a writer needs to actually draft the scene.

### Step 1: Clarify the Plot Point

Restate the input plot point. Identify:
- **Characters involved**
- **Setting** (where, when, environmental conditions)
- **Entry state** (how do characters feel coming into this scene?)
- **Exit state** (how should they feel leaving it?)

### Step 2: Identify the Scene Core

Every scene needs three things:

**Scene Goal** — What does the POV character want in this scene?
(Not the plot goal, the *scene-level* want. What are they trying to accomplish in the next 5 minutes?)

**Conflict** — What stands between them and that goal?
(Can be external obstacle, internal resistance, or interpersonal friction)

**The Turn** — How does the scene change the situation?
(New information, shifted relationship, altered stakes, decision made)

### Step 3: Break Into Beats

Expand the scene into 5-10 granular beats. Each beat is a micro-moment:

```
Beat 1: [Setup] — Establish where we are, what POV character is doing/feeling
Beat 2: [Inciting moment] — Something happens to kick off the scene's action
Beat 3: [Pursuit] — POV character moves toward their goal
Beat 4: [Obstacle] — Conflict manifests
Beat 5: [Escalation/Pivot] — Stakes rise or approach changes
Beat 6: [Climax] — The turn happens
Beat 7: [Resolution/New Status] — Aftermath, new equilibrium
```

Adjust beat count based on scene complexity. A quick transitional scene might have 3 beats; a major setpiece might have 12.

### Step 4: Output Format

```
## Beat Sheet: [Plot Point Summary]

### Scene Setup
- **Characters:** [list]
- **Setting:** [location, time, conditions]
- **Entry State:** [emotional/situational starting point]
- **Exit State:** [where we need to land]

### Scene Core
- **Goal:** [what POV character wants]
- **Conflict:** [what's in the way]
- **The Turn:** [how the scene changes things]

### Scene Beats

1. **[Beat Name]** — [Description of what happens]
   - *Tension level:* [1-5]
   - *Key sensory detail:* [one grounding image/sound/texture]

2. **[Beat Name]** — [Description]
   - *Tension level:* [1-5]
   - *Key sensory detail:* [...]

[Continue for all beats]

### Transition Out
[One sentence: how does this scene hand off to the next?]
```

### Notes

- Reference `Docs/outline.md` if it exists to understand where this scene fits in the larger structure
- Reference character sheets to inform emotional beats
- Flag if the plot point seems to be missing a clear Turn — every scene needs to change something
