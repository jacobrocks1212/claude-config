# Cognito Forms - Form Builder Skill

This skill provides deep expertise on the form builder feature. Activate this skill when working on the builder's UI, adding new field types, modifying its state, or changing how form definitions are saved.

## Core Concepts
- **Component Hierarchy**: The structure of the Vue.js components that make up the builder.
- **State Management**: How the form's definition is managed in the frontend as a user makes changes.
- **Field Type Integration**: The pattern for adding new field types to the builder.
- **Saving & Loading**: The process of serializing the form definition to send to the backend and deserializing it when loading an existing form.

## When to Reference Additional Files
- For a breakdown of the primary Vue components and their roles, read **`component-architecture.md`**.
- To understand how the form's state is managed (e.g., with Vuex or a similar pattern), read **`state-management.md`**.
